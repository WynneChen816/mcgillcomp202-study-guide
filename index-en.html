LESSON 1 · Week 1 Binary ↔ Decimal Conversion, Bits & Bytes, Encoding Formats 3 core concepts
💡

WHY DOES THIS LESSON MATTER?

This is the foundation of the whole course. Computers only understand 0s and 1s — every piece of text, image, and music is ultimately a sequence of binary digits. Understanding this helps you grasp why Python has so many different data types (int, float, str…) — the same string of bits becomes something completely different depending on how it's interpreted. Exams rarely ask heavy calculations, but concept and multiple-choice questions on this appear regularly.

Binary ↔ Decimal Conversion 📝 Memorise the method

Binary → Decimal (multiply each digit by its power of 2, then sum):

# Convert 10101100 to decimal: # Position: 7 6 5 4 3 2 1 0 ← count from the right, starting at 0 # Digit: 1 0 1 0 1 1 0 0 # # Calculation: 1×2⁷ + 0×2⁶ + 1×2⁵ + 0×2⁴ + 1×2³ + 1×2² + 0×2¹ + 0×2⁰ # = 128 + 0 + 32 + 0 + 8 + 4 + 0 + 0 # = 172 # So 10101100₂ = 172₁₀

Decimal → Binary (repeatedly divide by 2, collect remainders, read bottom-to-top):

# Convert 14 to binary: # 14 ÷ 2 = 7 remainder 0 ← least significant bit # 7 ÷ 2 = 3 remainder 1 # 3 ÷ 2 = 1 remainder 1 # 1 ÷ 2 = 0 remainder 1 ← most significant bit # # Read remainders bottom-to-top: 1 1 1 0 # So 14₁₀ = 1110₂
💡 ANALOGY Decimal counts in groups of 10 (0–9), carrying over when you hit 10. Binary only has 0 and 1, carrying over at 2 — like a light switch that is either ON or OFF. A row of switches together can represent any number.
Common powers of 2 (worth memorising!)Value

2⁷128
Bits, Bytes, and Encoding Formats 📝 Know the concepts
Bit A single 0 or 1 — the smallest unit of information in a computer.
Byte 8 bits grouped together.
1 Byte = 8 Bits.

The same binary string can mean completely different things depending on how it's interpreted:

# The binary number 1001000 could be interpreted as: # The number 72 # The character 'H' (ASCII encoding) # A colour in an image (RGB encoding) # # That's why we need standard formats to tell the computer what the bits mean: # .mp3 / .wav → audio # .jpeg / .png → image # ASCII / Unicode → text
⚠️ Common numeral systems (good to know) Decimal (base 10) · Binary (base 2) · Octal (base 8) · Hexadecimal (base 16, common in colour codes like #FF5733)
LESSON 2 · Week 2 Algorithm, Program, High/Low-Level Languages, print(), Comments 4 core concepts
📐

WHY DOES THIS LESSON MATTER?

This is the starting point of computational thinking. Before writing any code, you need to learn how to break a problem into clear, step-by-step instructions — that's what an algorithm is. You also need to understand what kind of language Python is, how it runs, and the most basic output tool: print(). The ideas here aren't hard, but they're fundamental to understanding what programming actually is.

Algorithm vs Program 📝 Know the difference
Algorithm A step-by-step plan for solving a problem, independent of any specific language. Can be expressed in natural language, pseudo-code, or flowcharts.
Program The actual implementation of an algorithm in a specific programming language (e.g. Python, Java, C++).
💡 ANALOGY An algorithm is a recipe written in words. A program is actually cooking the dish. The recipe can be translated into any language, but to actually make the food you must use a specific kitchen (programming language).
Low-Level vs High-Level Languages 📝 Know the categories
Low-LevelHigh-Level
ExamplesMachine code, AssemblyPython, Java, C++
ReadabilityPoor — close to 0/1 instructionsClose to human language
PortabilityTied to specific hardwareRuns on different machines
TranslationRuns directlyNeeds a Compiler or Interpreter

Python is a high-level language that uses an Interpreter — it translates and executes code line by line. That's why Python error messages can tell you the exact line number: the interpreter knows exactly where it crashed.

print(), \n, \t 📝 Know the usage ⭐ Must know
# Basic usage print("hello", "world") # hello world (default sep=" ") print("hello", "world", sep="-") # hello-world print("hello", end="") # no newline at end # Printing a variable x = 7 print("The value of x is", x) # The value of x is 7 # Special characters print("Line 1\nLine 2") # \n = newline print("col1\tcol2") # \t = tab (pads to next multiple of 4) # Comments: anything after # is ignored by Python x = 5 # this is an inline comment
LESSON 3 · Week 2 Variables, Types, input(), Arithmetic, Math Module, Expression vs Statement 5 core concepts
🔢

WHY DOES THIS LESSON MATTER?

Variables are the core of programming — they are your containers for storing data. This lesson teaches you the basic data types Python has, how to convert between them (type casting), how to get input from the user, and how to do arithmetic. Everything in later lessons builds on this.

Variables and Types 📝 Know the types ⭐ Must know
# Assignment: variable_name = value number = 67 name = "Alice" pi = 3.14 is_student = True # Check the type print(type(12.3)) # <class 'float'> print(type("hello")) # <class 'str'> print(type(5)) # <class 'int'> print(type(True)) # <class 'bool'>

Type Casting (converting between types):

int(3.14) # → 3 (truncates, does NOT round!) float(5) # → 5.0 str(42) # → "42" int("hello") # ❌ ValueError — can't convert text to int int("42") # ✅ → 42 (works if the string is a pure number)
⚠️ int() truncates — it does NOT round! int(3.9) gives 3, not 4. It just drops the decimal part. To round properly, use round(3.9).
input() Function ⭐ Must know 📝 Don't forget the trap

input() reads keyboard input from the user, but it always returns a string — even if the user types a number!

# ❌ Dangerous: forgot to cast the type x = input("Enter a number: ") print(x + 1) # TypeError! Can't add str + int # ✅ Correct: always cast x = int(input("Enter an integer: ")) print(x + 1) # Works fine now y = float(input("Enter a decimal: ")) print(y * 2)
💡 ANALOGY input() is like a translator who always replies in the same language (string), no matter what language you asked in. If you wanted a number, you have to translate it yourself (type cast).
Arithmetic Operators + Precedence 📝 Know the operators ⭐ Must know
10 + 3 # → 13 addition 10 - 3 # → 7 subtraction 10 * 3 # → 30 multiplication 10 / 3 # → 3.333... division (always returns float!) 10 ** 3 # → 1000 exponentiation (power) 10 // 3 # → 3 floor division (rounds down) 10 % 3 # → 1 modulus (remainder) # Operator precedence (high → low): # 1. () parentheses — highest # 2. ** exponentiation # 3. * / % // # 4. + - lowest
⚠️ The three most-confused operators / always returns a float (e.g. 6/2 = 3.0).
// floor division, rounds DOWN (e.g. 7//2 = 3, but -7//2 = -4!).
% gives the remainder (e.g. 7%3 = 1). Use it to check odd/even: n%2 == 0 means n is even.

Built-in functions + Math module:

# Built-in (no import needed) abs(-5) # → 5 absolute value pow(2, 10) # → 1024 same as 2**10 round(7.948, 2) # → 7.95 round to 2 decimal places # math module (must import first) import math math.sin(math.pi / 4) # sin(45°) math.cos(math.pi) # cos(180°) = -1.0 math.pi # → 3.141592653589793
Expression vs Statement 🧠 Understand the concept
Expression A piece of code that evaluates to a value.
E.g. 1/2, a + 5, "hello" + "world"
Statement A complete instruction that performs an action — doesn't necessarily return a value.
E.g. x = 5, print("hi")

A program executes statements top-to-bottom. When it encounters an expression, it evaluates it to a value and then moves on.

LESSON 4 · Week 3 Boolean, Relational Operators, and/or/not, if/elif/else 4 core concepts
🚦

WHY DOES THIS LESSON MATTER?

So far, programs just run from line 1 to the end with no decisions made. This lesson teaches you how to make your program take different paths depending on conditions — that's what makes programs "smart". Boolean values are the foundation for decisions, and if/elif/else is the gate that controls where the program goes next. Nearly every meaningful program uses conditional logic.

Boolean Values and Relational Operators 📝 Know the operators ⭐ Must know
# Relational operators: compare two values, result is True or False 7 == 7 # True (equal to) 7 != 5 # True (not equal to) 7 > 5 # True (greater than) 7 < 5 # False (less than) 7 >= 7 # True (greater than or equal to) 7 <= 6 # False (less than or equal to)
⚠️ Never confuse == and = = is assignment: x = 5 stores 5 in x.
== is comparison: x == 5 asks "is x equal to 5?" — result is True or False.
Writing if x = 5: causes an immediate SyntaxError. This is one of the most classic beginner bugs.

The float comparison trap:

x = 1.1 + 2.2 print(x == 3.3) # False!! Floating point has precision errors # Correct approach: check if they're "close enough" epsilon = 0.0001 print(abs(x - 3.3) < epsilon) # True ← this is the right way to compare floats
Boolean Operators: not, and, or 🧠 Understand the logic 📝 Know the truth tables
# not (negation): flips True ↔ False not True # False not False # True # and: True only if BOTH sides are True True and True # True True and False # False # or: True if AT LEAST ONE side is True True or False # True False or False # False # Precedence: not > and > or (like × > + in math) not True or False # = (not True) or False = False or False = False
💡 ANALOGY and: You need both a student card AND your homework to enter the lab — both conditions must be met.
or: You can pay with cash OR with your phone for bubble tea — one is enough.
not: Flips True/False, like "is NOT a student."
⚠️ Short-circuit evaluation True or (3 + "error") evaluates to True — because the left side is already True, Python never evaluates the right side (which would crash). Same with and: False and anything short-circuits to False immediately.
if / elif / else Conditional Statements ⭐ Must know 🧠 Understand the flow
# Basic structure if condition1: print("condition 1 is True") elif condition2: print("condition 2 is True") else: print("all conditions are False") # else is optional # Nested conditionals if x == y: print('x and y are equal') else: if x < y: print('x is less than y') else: print('x is greater than y')

Key rules:

  • Once one block executes, all remaining elif and else blocks are skipped
  • You can have as many elifs as you want
  • The final else is optional
  • Indentation is critical — Python uses it to determine which code belongs to which block
LESSON 5 · Week 4 Functions: def, Parameters, Scope, Void/Fruitful, Docstring, Common Mistakes 6 core concepts
⚙️

WHY DOES THIS LESSON MATTER?

Functions are the most important abstraction tool in programming. They let you wrap a reusable block of code, give it a name, and call it whenever needed — no copy-pasting. The concepts covered here — scope, return vs print, and docstrings — are among the highest-frequency exam and assignment topics in COMP202. Make sure you understand them thoroughly.

Defining (def) and Calling a Function ⭐ Must know 📝 Know the syntax
# Defining a function (defining ≠ running!) def greet(name): print("Hello, " + name) print("Welcome to McGill!") # Calling a function (must be AFTER the definition) greet("Alice") greet("Bob") # Returning multiple values (actually returns a tuple) def calculator(a, b): add = a + b subtract = abs(a - b) multiply = a * b divide = a / b return add, subtract, multiply, divide result = calculator(10, 5) print(result) # (15, 5, 50, 2.0)
⚠️ Defining a function does NOT run it! Writing def greet(): just tells Python "I'm defining this function." The body does not execute immediately. You must call greet() to actually run it. This is one of the most common beginner misconceptions.
Scope: Local vs Global Variables 🧠 Understand the logic ⭐ Must know
Local Variable Defined inside a function (including parameters). Only exists inside that function — gone when the function ends.
Global Variable Defined outside all functions. Functions can read it, but cannot modify it without the global keyword.
def my_function(): x = 5 # x is local — only exists inside this function print("inside:", x) X = 10 # X is global my_function() # inside: 5 print("outside:", X) # outside: 10 ← x and X are two completely separate variables # ❌ This crashes: trying to modify a global variable inside a function count = 5 def bad_function(): count += 7 # UnboundLocalError! Python sees it as an unassigned local variable
💡 ANALOGY A function is like a private workshop. The tools inside (local variables) only exist in there and disappear when you leave. The shared tools in the hallway (global variables) can be seen and used inside the workshop, but you can't throw them away or replace them without special permission (the global keyword).
Void vs Fruitful Functions: print vs return 🧠 Understand the logic ⭐ Very high-frequency exam topic
Void Function No return statement (or just return with no value). Actually returns None.
Uses print() to display output.
Fruitful Function Has a return value statement. The caller can store and use the returned value.
Uses return, not print.
# ❌ Common mistake: print is NOT the same as return def bad_add(a, b): print(a + b) # only displays — returns nothing result = bad_add(3, 4) print(result) # None ← nothing was returned! # ✅ Correct: use return def good_add(a, b): return a + b result = good_add(3, 4) print(result) # 7 ← correct # return immediately exits the function — code after it won't run def early_exit(x): if x < 0: return 0 # exits here; the next line never runs return x * 2
⚠️ Always catch the return value! Calling test() without assigning the result means the return value vanishes immediately. Write value = test() to hold onto it.
Docstring Format 📝 Memorise the format ⭐ Must know

A docstring is written right after the function header, wrapped in triple quotes. It's a COMP202 assignment standard — wrong format costs marks!

def area_circle(radius): """ (num) -> num Returns the area of a circle with a given radius. >>> area_circle(20) 1256.64 >>> area_circle(0) 0 """ import math return math.pi * pow(radius, 2.0)

Three components of a docstring:

  • Type contract: first line, format (input types) -> return type
  • Description: what the function does
  • Examples: lines starting with >>> showing inputs and expected outputs (these become doctests in L13!)
LESSON 6 · Week 5 While Loop: Syntax, Update Condition, Infinite Loops, Shorthand Assignment 4 core concepts
🔄

WHY DOES THIS LESSON MATTER?

A while loop makes your program repeat something. Unlike a for loop (L7, used to traverse a known sequence), a while loop is for "I don't know how many times, but I know when to stop" — e.g., keep asking the user to input until they give a valid answer. Paired with try/except from L19, this is the classic pattern for handling user input errors.

While Loop Syntax + Execution Flow 🧠 Understand the logic ⭐ Must know
# Basic syntax while condition: # condition is checked before every iteration # ⚠️ something inside MUST eventually make condition False! # Full example: countdown n = 3 while n > 0: print('Value of n is:', n) n = n - 1 # ← critical: update n so the condition eventually becomes False # Output: 3, 2, 1 (when n=0, 0>0 is False — loop stops)
💡 ANALOGY A while loop is like a washing machine cycle: "keep washing as long as the clothes aren't clean." After each wash it checks — still dirty? wash again. Clean? Stop. The key is: the washing process must be making the clothes progressively cleaner, otherwise it will wash forever (infinite loop).
Infinite Loop — 3 Common Causes 🧠 Know why it happens ⭐ Must avoid
# ❌ Cause 1: condition is never updated n = 3 while n > 0: print('hello') # n stays 3 forever # ❌ Cause 2: updated in the wrong direction n = 6 while n > 5: n = n + 1 # n keeps growing — always > 5 # ❌ Cause 3: condition reset to a constant every iteration n = 6 while n > 5: n = 7 # n is always reset to 7 — always > 5

If you get stuck in an infinite loop in Thonny, press Ctrl + C to force-stop the program.

Shorthand Assignment Operators 📝 Know the shorthands
x += 3 # same as x = x + 3 x -= 3 # same as x = x - 3 x *= 3 # same as x = x * 3 x /= 3 # same as x = x / 3 x **= 2 # same as x = x ** 2 x %= 3 # same as x = x % 3 x //= 3 # same as x = x // 3 # Used in while loops to update a counter concisely count = 0 while count < 5: print(count) count += 1 # cleaner than count = count + 1
LESSON 7 · Week 5 Recursion, For Loop, Variable Scope, Break/Continue 4 core concepts
🏗️

WHY DOES THIS LESSON MATTER?

The core of this lesson is "making your program repeat work efficiently." For loops let you process data in bulk; recursion gives you an alternative problem-solving style; and understanding variable scope tells you exactly when a variable "exists" and when it "disappears." These three things are the foundation of writing genuinely useful programs.

Recursion 🧠 Understand the logic ⭐ Must know

Recursion is a function calling itself. Every recursive function needs exactly two things — miss either one and it breaks:

  • Base case: tells the program when to stop — without this, you get infinite recursion and a crash
  • Recursive step: calls itself, but must get one step closer to the base case each time
def countdown(n): if n <= 0: # ← Base case: stop here! print('Blastoff!') else: print(n) countdown(n - 1) # ← Recursive step: n decreases each time
💡 ANALOGY Imagine counting a stack of plates. You pick up the top plate, count "1," and ask the person next to you to count the rest — and they do the same thing. Eventually the last person finds no plates left (base case), reports "0," and the answer bubbles back up through everyone. That's how recursion works.

Recursion vs loop — when to use which?

  • Many iterations, care about memory and speed → use a loop (while or for)
  • Not performance-critical, want cleaner code for naturally recursive problems → use recursion
For Loop + range() 📝 Memorise ⭐ Must know
for variable in sequence: # sequence = something iterable (list, range, string…) instructions # runs once per element, must be indented
💡 ANALOGY Like a teacher taking attendance: they go through the list (sequence), call each name (variable gets assigned that student's name), the student answers (loop body runs), then on to the next.

Three ways to use range() — must memorise!

SyntaxNumbers generatedHow to remember
range(5)0, 1, 2, 3, 4From 0 up to (not including) end
range(3, 7)3, 4, 5, 6From start up to (not including) end
range(10, 2, -2)10, 8, 6, 4Step can be negative — counts down
⚠️ Extremely common mistake range() never includes the right endpoint. This is called a half-open interval, written [start, end). Many people forget this and end up one number short.

For loop or while loop?

  • For: you know how many times to iterate, or you want to process each element in a sequence
  • While: you don't know how many times, just a stopping condition (e.g. keep re-prompting until valid input)
Variable Scope 🧠 Understand the logic

In Python, the smallest "local" scope is a function body — not an if/for/while block! This means variables defined inside an if or for block are accessible outside it (as long as that branch actually ran).

x = 5 if x > 0: y = x print(y) # ✅ Fine! The if branch ran, so y exists → prints 5 x = 5 if x > 0: y = x else: z = -x # x=5 > 0, so the else never ran print(z) # ❌ NameError! z was never created
💡 ANALOGY A variable only exists if the code that creates it actually ran. If that if/else branch was never executed, the variable was never "placed" in memory — trying to access it is like reaching into an empty box.
break and continue 📝 Know the difference
KeywordEffectAnalogy
breakImmediately exits the entire loop — all remaining iterations cancelledSkipping class entirely — you walk out the door
continueSkips the rest of this iteration — jumps straight to the next oneSleeping through one class — you're still in your seat, but you miss this one and wait for the next
for i in range(1, 11): if i == 5: break print(i, end=" ") # Output: 1 2 3 4 ← hit 5, exits immediately, rest gone for i in range(1, 11): if i == 5: continue print(i, end=" ") # Output: 1 2 3 4 6 7 8 9 10 ← skipped 5, everything else printed
LESSON 8 · Week 6 Random, Modules, Strings Basics 4 core concepts
🔧

WHY DOES THIS LESSON MATTER?

This lesson opens the door to "using tools that others have already written" (modules), and introduces one of the most common data types — strings. Strings appear in virtually every program, so understanding their underlying rules (immutability, indexing, etc.) is essential for everything that follows.

Modules — 4 Import Styles 📝 Know the syntax
SyntaxHow to callBest used when
import mathmath.sqrt(16)Safest — no naming conflicts
from math import sqrtsqrt(16)Only need a few functions — cleaner syntax
from math import *sqrt(16)Use carefully! May clash with your own names
import math as mm.sqrt(16)Long module name — use an alias
💡 ANALOGY import math = borrowing a whole textbook — you must look up every function in the index (math.xxx). from math import sqrt = photocopying just that one page and putting it in your notes. from math import * = copying the whole book into your notes — risky if you already have something called "sqrt" in there.

__name__ and __main__ 🧠 Understand the logic

When a .py file is run directly, Python sets __name__ to "__main__". When it's imported by another file, __name__ becomes the filename instead. Use if __name__ == "__main__": at the bottom of a module so certain code only runs when the file is executed directly, not when it's imported.

Random Module 📝 Know the API names
import random random.random() # random float in [0.0, 1.0) — excludes 1.0 random.randint(2, 5) # random int from 2 to 5 INCLUSIVE on both ends! random.seed(123) # fix the seed to make results reproducible
⚠️ Easy to confuse randint(2, 5) is inclusive on both ends — 2 and 5 can both appear. But range(2, 5) excludes 5. Same-looking format, different rules — don't mix them up!

random.seed(): with the same seed, every run produces the same "random" sequence. Extremely useful for testing, since you can reproduce your results.

Strings — Indexing, Slicing, Methods 📝 Memorise ⭐ Must know

Indexing: first character is [0], last is [-1]

fruit = 'banana' fruit[0] # 'b' fruit[-1] # 'a' (negative index counts from the end) fruit[1000] # ❌ IndexError (out of range)

Slicing: s[start : end : step] — incredibly useful

s = 'Hello world' s[0:5] # 'Hello' (index 0 to 4, excludes 5) s[6:] # 'world' (from 6 to end) s[0:5:2] # 'Hlo' (step=2, take every other character) s[::-1] # 'dlrow olleH' (reverse the string — classic trick!)

Common string methods:

s.lower() / s.upper()Convert to all lowercase / all uppercase — returns a new string
s.replace(old, new)Replace all occurrences of old with new
s.count(c)Count how many times c appears
s.find(c)Find the first index of c; returns -1 if not found
⚠️ Most important string property: strings are immutable! Strings are immutable. Every method returns a brand new string — the original is never modified. You can't do s[0] = 'X' — that throws a TypeError.
💡 ANALOGY Strings are like words carved into stone. You can't chisel out one letter and replace it — you have to carve a brand new stone (create a new string).
LESSON 9 · Week 7 Three Ways to Traverse a String 3 core patterns
🔍

WHY DOES THIS LESSON MATTER?

Processing strings character-by-character is one of the most fundamental operations in programming. These three traversal patterns are templates you'll reuse constantly — in assignments, interview problems, and real projects. Each suits a different scenario; knowing when to pick which is the skill.

The 3 Traversal Patterns 📝 Know all three patterns ⭐ Must know
s = "banana" # Pattern 1: for-in loop (simplest — just need the character) for char in s: print(char) # b, a, n, a, n, a # Pattern 2: for loop with range (need the index too) for i in range(len(s)): print(i, s[i]) # 0 b, 1 a, 2 n, ... # Pattern 3: while loop (most flexible — can skip ahead or go backwards) i = 0 while i < len(s): print(s[i]) i += 1
PatternBest forCan get index?
for char in sJust need each character❌ No (use enumerate if needed)
for i in range(len(s))Need both index and character✅ Yes
while i < len(s)Need flexible control (skip, jump)✅ Yes
Three Classic Loop Patterns 🧠 Understand when to use each
# Reducing (reduce many values to one) total = 0 for char in "hello": total += 1 print(total) # 5 (count characters) # Mapping (transform each element — same count, different values) result = "" for char in "hello": result += char.upper() print(result) # "HELLO" # Filtering (keep only elements matching a condition) vowels = "" for char in "hello": if char in "aeiou": vowels += char print(vowels) # "eo"
💡 ANALOGY Reducing = juicing a bag of apples into one glass (many → one). Mapping = peeling each apple (same count, each one transformed). Filtering = picking out only the good apples and discarding the bad ones (fewer remain).
LESSON 10 · Week 7 List: Creation, Access, Operations, Built-in Functions 5 core concepts
📋

WHY DOES THIS LESSON MATTER?

Lists are arguably the most used data structure in Python. Wherever you need to store a collection of things and process them one by one, you'll use a list. Every lesson from here on uses lists — they're non-negotiable.

List Basics: Creation and Access ⭐ Must know 📝 Memorise
# Creating a list fruits = ['apple', 'banana', 'cherry'] mixed = [1, "hello", True, 3.14] # can hold any types empty = [] # Accessing elements (same indexing as strings) fruits[0] # 'apple' (first element) fruits[-1] # 'cherry' (last element) fruits[1:3] # ['banana', 'cherry'] (slicing works too) # Lists are MUTABLE — you can change elements fruits[0] = 'avocado' print(fruits) # ['avocado', 'banana', 'cherry']
List Methods Quick Reference 📝 Memorise names and effects ⭐ Must know
lst = [3, 1, 4, 1, 5] lst.append(9) # adds 9 to the end → [3,1,4,1,5,9] lst.insert(2, 99) # inserts 99 at index 2 → [3,1,99,4,1,5,9] lst.remove(1) # removes first occurrence of 1 lst.pop() # removes and returns last element lst.pop(0) # removes and returns element at index 0 lst.sort() # sorts in place (modifies the list!) lst.reverse() # reverses in place lst.count(1) # how many times 1 appears lst.index(4) # index of first occurrence of 4 # Built-in functions that work on lists len(lst) # number of elements min(lst) # smallest element max(lst) # largest element sum(lst) # sum of all elements (numbers only)
⚠️ sort() modifies the list in place — it returns None! new_list = lst.sort() gives you None. If you want a sorted copy without touching the original, use new_list = sorted(lst) instead.
LESSON 11 · Week 7 Nested Loops, Nested Lists, String ↔ List Conversion 4 core concepts
🗂️

WHY DOES THIS LESSON MATTER?

Real-world data is often 2D: spreadsheets, matrices, image pixels… Nested lists + nested loops are the core pattern for handling this. split/join are indispensable text-parsing tools used in almost every real-world program.

💡 ANALOGY Like searching for a parking spot in a multi-storey car park: the outer loop is "which floor," the inner loop is "check every spot on this floor." For every one step of the outer, the inner runs a full lap.

matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]] matrix[1] # [4, 5, 6] ← second row matrix[1][2] # 6 ← second row, third column # Rule: matrix[row][column] — row first, then column

LESSON 12 · Week 8 Object Identity, Aliasing / Shallow / Deep Copy, Mutable vs Immutable 4 core concepts
🧬

WHY DOES THIS LESSON MATTER?

This is Python's "under the hood" lesson. Understanding references and copying prevents a particularly sneaky class of bug: you think you're modifying a "copy," but you're actually changing the original, and your data mysteriously changes elsewhere. This is frequently tested in assignments and interviews.

Object Identity — is vs == 🧠 Understand the logic

Every Python object has a unique memory address, viewable with id().

  • ==: checks whether two variables have the same value
  • is: checks whether two variables point to the exact same object in memory (equivalent to id(a) == id(b))
a = "hello" b = "hel" b += "lo" a == b # True ← same value a is b # False ← different objects (different memory addresses) a = 400 b = a # b points to the same object as a a is b # True b = 200 # b now points to a new object a is b # False
💡 ANALOGY == checks if two people have the same name. is checks if they're literally the same person (same ID card). Two different people can both be named "Alex" (== True) but are still two distinct people (is False).
Aliasing, Shallow Copy, and Deep Copy — What's the Difference? 🧠 Understand the logic ⭐ Critical — high exam frequency

These three are completely different things. The clearest way to understand them is to split by 1D list vs 2D list (nested list).

① Aliasing — this is NOT a copy at all!

list2 = list1 doesn't create a new list. It just makes list2 point to the same block of memory. Two names, one object.

list1 = [1, 2, 3] list2 = list1 # ← Aliasing! list2 is NOT a new list — just another name for the same one list1[0] = 99 print(list1) # [99, 2, 3] print(list2) # [99, 2, 3] ← list2 changed too! They were the same thing all along print(list1 is list2) # True ← confirmed: identical memory address
💡 ANALOGY Aliasing is like giving two nicknames to the same person — "Alex" and "Al" refer to the same individual. Tell "Alex" to get a haircut, "Al" will have shorter hair too.

② 1D list: .copy() or [:] is enough

For a flat list whose elements are immutable values (ints, strings…), .copy() and [:] create a genuinely independent new list. Modifying one won't affect the other.

list1 = [1, 2, 3] list2 = list1.copy() # method 1: .copy() list3 = list1[:] # method 2: slicing — exactly the same effect print(list1 is list2) # False ← now they are two separate list objects list1[0] = 99 print(list1) # [99, 2, 3] print(list2) # [1, 2, 3] ← unaffected ✅ print(list3) # [1, 2, 3] ← also unaffected ✅

③ 2D list: .copy() is NOT enough — you must use copy.deepcopy()

This is where the most hidden bugs live! When a list contains other lists, .copy() only duplicates the outer shell — the inner sub-lists are still shared references. Modify an inner element and both copies change.

import copy original = [[1, 2], [3, 4]] # ⚠️ .copy() on a 2D list = shallow copy — outer is new, inner is still shared shallow = original.copy() print(original is shallow) # False ← outer list IS new print(original[0] is shallow[0]) # True ← but [1,2] is the SAME object! original[0][0] = 99 # modify the inner sub-list print(original) # [[99, 2], [3, 4]] print(shallow) # [[99, 2], [3, 4]] ← shallow changed too! 😱 # ✅ deepcopy() duplicates every level independently deep = copy.deepcopy(original) print(original[0] is deep[0]) # False ← inner sub-lists are also new objects original[0][0] = 0 print(original) # [[0, 2], [3, 4]] print(deep) # [[99, 2], [3, 4]] ← deep copy completely unaffected ✅
💡 ANALOGY — using apartments Aliasing: You give your friend your address. They move into YOUR apartment. Whatever they rearrange, you feel it.

Shallow copy (works for 1D, fails for 2D): You move into a brand new apartment — but all the furniture inside is rented shared furniture. Your friend rearranges the shared sofa in their place, and yours disappears too.

Deep copy: You get a brand new apartment AND all-new furniture bought fresh. Your friend can redecorate theirs however they like — yours is completely independent.

⚠️ Decision rule — use this flow every time Is your list 1D (elements are numbers, strings)? → .copy() or [:] is sufficient.
Is your list 2D or deeper (lists inside lists)? → You MUST use copy.deepcopy() or the inner lists will still be shared.
Mutable vs Immutable 📝 Know the categories
✅ Mutable List, Dictionary
Content can be changed in-place; same id
❌ Immutable String, int, float, bool, tuple

Important implication: when you pass a list to a function, the function receives a reference to it. If the function modifies the list, the original list outside also changes! This does not happen with int or string.

LESSON 13 · Week 8 Doctest, Debugging, Raising Exceptions 3 core concepts
🛡️

WHY DOES THIS LESSON MATTER?

Writing code has two steps: making it run, and making it run correctly. All previous lessons taught the first step. This lesson teaches the second — how to test code, find bugs, and how to "reject" bad input gracefully and tell the caller what went wrong. In large projects, test and error-handling code can be just as voluminous as the feature code itself. This lesson is the beginning of writing professional-grade code.

Doctest 📝 Memorise the format ⭐ Must know

Think of doctest as an "automated taste-test" system. When you write a function, you include a few "if I pass this in, this should come out" examples in the docstring. Python will automatically verify them.

Two lines to activate doctests:

import doctest # line 1: import the module doctest.testmod() # line 2: run all tests (put this at the bottom of your file)

How to write doctests: inside the docstring, lines starting with >>> are test inputs, the next line is the expected output:

def add(x, y): """ (num, num) -> num Returns the sum of x and y. >>> add(2, 2) 4 >>> add(-2, 3) 1 >>> add(1.2, 3.5) 4.7 """ return x + y import doctest doctest.testmod() # all pass → nothing printed (silence = good!)
💡 ANALOGY >>> add(2, 2) is you saying "test: put 2 and 2 in, machine should output 4." If your function accidentally uses multiplication instead of addition, add(-2, 3) exposes the bug — it should be 1 but multiplication gives -6. Good doctests must cover edge cases, not just the "normal" case.
⚠️ Doctest is character-for-character strict If your function returns 4.699999999999999 (floating point precision) but your docstring says 4.7, the test fails. Solution: use round() on the result, or write the expected value in the exact form it will appear.

When your function raises an exception, doctest for it like this:

def print_multiples(n, times): """ >>> print_multiples('2', 4) Traceback (most recent call last): ... TypeError: the function expects two integers as input """ if not isinstance(n, int) or not isinstance(times, int): raise TypeError("the function expects two integers as input")

Debugging — 4 Types of Errors 📝 Know the categories 🧠 Understand the logic
① Stylistic Errors Code runs fine but is hard to read. E.g. messy variable names, inconsistent indentation.
Not a real error — but may cost marks!
② Syntax Errors Invalid code structure — Python refuses to run at all. Detected first.
Fix these before anything else.
③ Runtime Errors Syntax is fine, but the program crashes during execution. E.g. dividing by zero, wrong type.
Discovered one step later than syntax errors.
④ Logic Errors Program finishes without crashing, but the output is wrong. Python never complains.
Hardest to find — entirely up to you.
# ❌ Syntax Error — missing colon x = int(input("Enter a number: ")) if x > 100 # missing ':' → SyntaxError, program won't run at all print("You think big!") # ❌ Runtime Error — wrong type x = input("Enter a number: ") # input() returns str! if x > 100: # TypeError: '>' not supported between str and int print("You think big!") # ❌ Logic Error — program runs, but result is wrong x = input("Enter your favourite number: ") print("Multiplied by 2:", x * 2) # Input 5 → outputs "55" because "5" * 2 = "55" (string repetition, not math!)
💡 ANALOGY Syntax Error: You write a letter with grammatical errors so bad the secretary returns it without even sending it in.
Runtime Error: The letter gets sent, but the system crashes because you put the wrong ID number format.
Logic Error: Everything processes successfully, but you wrote the wrong destination — you end up in the wrong city and no one told you.
⚠️ Syntax errors block all other error detection If there's a syntax error, Python won't run at all, so runtime and logic errors stay hidden. Always fix syntax errors first, then re-run and look for the next type of error.

How to fix each type:

  • Syntax / Runtime: read the error message carefully — it tells you the line number and what went wrong. Search the course slides or online.
  • Logic (hardest!): add print() statements at suspicious spots to trace how variables change; use the IDE debugger to step through line-by-line; try rubber ducking — explain the code out loud to an inanimate object (a rubber duck, a teddy bear, anyone). You'll often spot the bug mid-explanation.
Raising Exceptions — Deliberately Signalling Errors 🧠 Understand the logic 📝 Know the syntax ⭐ Must know

An exception is an event that disrupts the normal flow of a program. Python raises some automatically (like TypeError, ZeroDivisionError). But as a programmer, you can — and should — raise exceptions yourself to signal that invalid input was received.

💡 ANALOGY You run a food delivery app. Someone orders "0 servings of fried rice." The system won't crash, but that order makes no sense. As the programmer, you should proactively reject it and say "must order at least 1 serving" — rather than silently processing an empty order. That's what raise Exception does: catch bad input early, make bugs easier to find.

Syntax:

raise SomeExceptionType("Error message goes here") # Common exception types: raise TypeError("message") # wrong type passed in raise ValueError("message") # right type, but value is invalid

Complete example:

def divide(a, b): """ (int, int) -> float Divides a by b. >>> divide(10, 2) 5.0 >>> divide(10, 0) Traceback (most recent call last): ... ValueError: The divisor 'b' cannot be zero. >>> divide(10.0, 2) Traceback (most recent call last): ... TypeError: Both arguments must be integers. """ if not isinstance(a, int) or not isinstance(b, int): raise TypeError("Both arguments must be integers.") if b == 0: raise ValueError("The divisor 'b' cannot be zero.") return a / b
Exception TypeWhen to raiseExample
TypeErrorWrong type passed inFunction expects int, got str
ValueErrorRight type, but value is invalidFunction expects positive number, got -5
ZeroDivisionErrorDividing by zeroUsually Python raises this automatically; you can also raise it early
⚠️ Don't confuse TypeError vs ValueError You ordered a "bubble tea" and got a plate of fried rice — that's TypeError (completely wrong type). You ordered a "less sweet" bubble tea and got it extra sweet — that's ValueError (right type, wrong value).

raise and doctest relationship: if your function uses raise, you must add corresponding exception doctest examples — see the format above. This is a common mark-deduction point in COMP202 assignments.

Lists store things and you find them by position (index). Dictionaries let you attach a "label" (key) to each item, so you retrieve it by name — no need to know its position. Think of it as upgrading from a bookshelf to a library catalogue system. For large datasets, dict lookups are dramatically faster than list searches. Dicts appear constantly in real projects.

Creating a Dictionary + Key/Value Rules 📝 Know the rules ⭐ Must know

Each element in a dictionary is called an item, made up of a key and a value, separated by :.

# Method 1: curly braces {} heights = {'Sasha': 182, 'Rohan': 168, 'Kate': 177} # Method 2: dict() function — creates an empty dict empty_dict = dict() also_empty = {} # both work

Must be unique within a dict (no duplicates).
✅ Value rules Can be any type, including list, dict, None.
Does not need to be unique.

Accessing, Adding, Removing 📝 Know the syntax
heights = {'Sasha': 182, 'Rohan': 168, 'Kate': 177} # Access a value: dict[key] print(heights['Rohan']) # 168 # Add / modify: simple assignment heights['Rohan'] = 175 # key exists → overwrites old value heights['Lena'] = 165 # key doesn't exist → adds new item # Delete: del dict[key] del heights['Sasha'] print(heights) # {'Rohan': 175, 'Kate': 177, 'Lena': 165}

Classic Pattern: Counting Occurrences with a Dict 🧠 Understand the logic ⭐ Must know

This is the most frequently tested practical dict application in COMP202. The idea: key = "what the thing is," value = "how many times it appeared."

def count_occurrences(input_list): """ (list) -> dict Returns a dict counting how many times each element appears. >>> count_occurrences(['a', 'b', 'a', 'c', 'b', 'a']) {'a': 3, 'b': 2, 'c': 1} """ count = {} for element in input_list: if element in count: count[element] += 1 # seen before — increment else: count[element] = 1 # first time — initialise to 1 return count

File Operations: Open → Read/Write → Close 📝 Know the workflow ⭐ Must know
💡 ANALOGY Working with files is like borrowing a library book: first check it out (open), then read or write notes (read/write), and finally return it (close). Not returning the book (not closing the file) locks it up for others and may cause data loss.

open() mode parameter:

Reading a file:

# Method 1: for loop line-by-line (most common — memory-friendly) fobj = open('quotes.txt', 'r') for line in fobj: print(line) # note: each line ends with '\n' fobj.close() # Method 2: read entire file into one string fobj = open('quotes.txt', 'r') content = fobj.read() fobj.close() # Method 3: read into a list, one line per element fobj = open('quotes.txt', 'r') lines = fobj.readlines() # → ['line1\n', 'line2\n', ...] fobj.close()

# Overwrite (clears existing content!) fobj = open('output.txt', 'w') fobj.write('Hello, world!\n') # write() does NOT add \n automatically fobj.write('Second line\n') fobj.close() # Append (preserves existing content) fobj = open('output.txt', 'a') fobj.write('This is appended.\n') fobj.close()

2. 'w' mode immediately erases the existing file — unrecoverable. If you want to keep original content, use 'a'.
Pickle Module 📝 Know the syntax

LESSON 19 · Week 12 Exception Handling: Try/Except, Multiple Exceptions, Finally 3 core concepts
🪤

WHY DOES THIS LESSON MATTER?

L13 taught you to "raise (throw) exceptions" — announcing that something is wrong. This lesson teaches you to "catch exceptions" — when an error occurs, instead of crashing, you handle it gracefully. Imagine a program that asks the user to type a number. You can't stop users from typing "abc" — but you can build in a safety net so the program doesn't crash; instead it says "please try again." That's try/except.

Try / Except — Basic Structure 🧠 Understand the logic ⭐ Must know
💡 ANALOGY Try/except is like an airbag in a car:
try block = driving normally (doing what you want to do)
except block = airbag deploys on crash (what to do when an error occurs)
With an airbag, a crash doesn't necessarily kill you. With except, an error doesn't necessarily crash your program.
try: s = input('Please enter a number: ') number = int(s) # ← this might crash (if user types "abc") print('No error, you entered:', number) except ValueError: print('That was not a valid number!') # ← only runs if ValueError occurs

Flow: Python runs the try block. If a ValueError is raised, it jumps straight to the except ValueError block. If no error occurs, the except block is skipped entirely.

⚠️ When should you use try/except? Only for errors you CAN'T prevent as a programmer! Things like user input, network connections, files that may or may not exist — you can't control these. But never use try/except to "hide" bugs in your own code — that makes bugs unfindable.
Handling Multiple Exceptions 📝 Know the syntax
# Method 1: separate except blocks (recommended — handle each differently) try: result = int(input('Enter index: ')) my_list = [10, 20, 30] print(my_list[result]) except ValueError: print('Input is not a number') except IndexError: print('Index out of range') # Method 2: tuple to catch multiple in one except try: pass except (IndexError, TypeError): print('IndexError or TypeError occurred') # Method 3: bare except — catches ALL errors (use sparingly!) try: pass except: print('Something went wrong') # hides what kind of error — use with caution
finally Block — Always Executes 🧠 Understand the logic

Code inside finally always runs — whether the try succeeded, whether except was triggered, even if there's a return, break, or continue inside the try/except blocks.

while True: try: s = input('Please enter a valid number: ') number = int(s) break # success — exit the while loop except ValueError: print('Invalid input!') finally: print('This always prints.') # runs every iteration, success or failure print("Valid number entered: " + str(number))
💡 ANALOGY finally is like "clean up the tables before closing the restaurant" — no matter how the day went, no matter if customers complained or loved it, you always clean up before closing. In file operations, finally is commonly used to guarantee close() always gets called, even if something went wrong during reading.
LESSON 20 · Week 13 OOP 1: Object-Oriented Programming, Objects, Classes, Attributes 4 core concepts
🏛️

WHY DOES THIS LESSON MATTER?

Until now, all data types you used (int, list, str) were Python built-ins. OOP (Object-Oriented Programming) lets you define entirely new data types. This is a major upgrade: you can create a "Student" type that stores a name, student ID, and grades, and that has its own operations like "get grades" and "change name" — far cleaner than managing everything in separate lists and dicts. Almost all large-scale software is written using OOP.

What is Object-Oriented Programming? 🧠 Understand the concept
💡 ANALOGY Think of LEGO. The factory designs a mould (Class/Blueprint) for a mini-figure that specifies it has a head, body, arms, and legs, and can "sit" and "hold things." The factory uses this mould to produce unlimited individual figures — each figure is an Object (instance). Each figure's colour and hair can differ, but all follow the same mould.

In Python: Class = the mould, Object = a specific thing made from the mould.

You've been using OOP since day one — you just didn't know it:

a = "hello" # a is an object (instance) of the str class a.upper() # upper() is a method defined in the str class my_list = [1, 2, 3] # my_list is an object of the list class my_list.append(4) # append() is a method defined in the list class print(type(a)) # <class 'str'> ← everything belongs to some class!
Defining a Class 📝 Know the syntax ⭐ Must know
# Define a new class (a blueprint) class MyClass: """a new type""" # Create an object (an instance) from the class my_object = MyClass() # call the class name like a function print(type(my_object)) # <class '__main__.MyClass'> ← '__main__' means the current file

Key distinction:

  • Class: the blueprint / design — not a concrete thing itself
  • Object / Instance: a concrete thing created from the class — MyClass() creates one
⚠️ Class names must start with a capital letter! Python convention: class names use CamelCase (e.g. Student, BankAccount); variable and function names use snake_case (e.g. my_student, get_balance). This is a basic code readability standard.
Attributes — Attaching Data to Objects 🧠 Understand the logic ⭐ Must know

An attribute is a "variable that belongs to this specific object." Access and modify via object.attribute_name:

class MyClass: """a new type""" my_object = MyClass() # Set attributes my_object.name = "John" my_object.age = 20 # Read attributes print(my_object.name) # John print(my_object.age) # 20 # Modify attributes my_object.name = "Max" print(my_object.name) # Max
💡 ANALOGY Attributes are like fields on a personal profile: name, age, student ID. Each object has its own set, completely independent of other objects. my_object.name means "the name field belonging to this specific object" — the dot means "belonging to."

WHY DOES THIS LESSON MATTER?

Now that you know what a class is, this lesson teaches you how to write one properly: use __init__ so every object starts with its own data; use methods to give objects things they can do; use __str__ to control what an object looks like when printed. After these two OOP lessons, you'll understand exactly why my_list.append() works the way it does — and you'll be able to build your own types that work the same way.

Constructor: __init__ 🧠 Understand the logic ⭐ Must know

Every time you write MyClass() to create an object, Python automatically calls a special method named __init__ — that's the constructor. You define it yourself to control what data the object starts with:

class Student: def __init__(self, name, ID=0): # self represents "the object currently being created" self.name = name # store the passed-in name as an attribute self.ID = ID # ID has a default value of 0 # Creating objects — arguments get passed to __init__ john = Student("John", 20) print(john.name, john.ID) # John 20 # With default value, can pass fewer arguments default_student = Student("no name") print(default_student.name, default_student.ID) # no name 0
💡 ANALOGY __init__ is like the registration form new students fill in on their first day. Every student (object), the moment they enrol (are created), must provide their name and student ID. Those details are then permanently associated with that person (object) and can be looked up any time.

Key things to know about self:

  • self is a convention (you could name it anything, but never do)
  • self represents "this specific object." Inside a class, you don't know what the object will be named later, so you use self as a stand-in
  • self must be the first parameter in __init__, but you don't pass it when calling — Python inserts it automatically
  • __init__ should have no return statement
⚠️ Forgetting required arguments raises TypeError If __init__(self, name, ID) has no defaults, calling Student() with no args raises TypeError: __init__() missing 2 required positional arguments. Note: self doesn't count!
Class Attributes vs Instance Attributes 🧠 Understand the logic ⭐ Easy to confuse!
Instance Attribute Defined with self.xxx inside __init__.
Each object has its own independent copy.
Class Attribute Defined at the top of the class (outside any method).
Shared across ALL objects of that class.
class Car: fuel = 100 # ← Class Attribute: shared by all Car objects def __init__(self, brand): self.brand = brand # ← Instance Attribute: unique to each car car1 = Car("Toyota") car2 = Car("BMW") print(car1.fuel) # 100 ← inherited from class print(car2.fuel) # 100 car1.fuel = 20 # creates a new INSTANCE attribute for car1 — shadows the class attribute print(car1.fuel) # 20 ← car1's own instance attribute print(car2.fuel) # 100 ← car2 is unaffected — still sees the class attribute
⚠️ To truly change the class attribute for all objects, use the class name car1.fuel = 50 only creates a new instance attribute for car1 — it doesn't modify the class attribute. To update it for everyone: Car.fuel = 50.
Methods — Functions That Belong to a Class 🧠 Understand the logic ⭐ Must know

A method is just a function defined inside a class. The only difference from a regular function: the first parameter must be self (the object calling the method).

class Student: def __init__(self, name, ID=0): self.name = name self.ID = ID def print_info(self): # only self — no extra params print("Name:", self.name, "ID:", self.ID) def modify_name(self, new_name): # self + extra param self.name = new_name # modifies THIS object's name john = Student("John", 20) john.print_info() # Name: John ID: 20 john.modify_name("John Doe") john.print_info() # Name: John Doe ID: 20

When you call john.print_info(), Python automatically passes john as self behind the scenes — equivalent to Student.print_info(john).

__str__() — Control What print() Shows 📝 Know the usage ⭐ Must know

By default, printing a custom object shows a useless memory address:

john = Student("John", 20) print(john) # <__main__.Student object at 0x0000024F6EBC8F10> ← meaningless

Define __str__ to make print show something useful:

class Student: def __init__(self, name, ID=0): self.name = name self.ID = ID def __str__(self): # Must RETURN a string — do NOT print inside __str__! return "Student: " + self.name + ", ID: " + str(self.ID) john = Student("John", 20) print(john) # Student: John, ID: 20 ← now it looks like a person! print(str(john)) # same — str() also calls __str__
💡 ANALOGY Everyone introduces themselves differently depending on context. __str__ teaches this object "when someone prints you, this is how you introduce yourself." Without it, you blurt out your memory address. With it, you say "I'm John, student ID 20."
