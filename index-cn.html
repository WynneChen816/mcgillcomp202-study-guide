LESSON 1 · Week 1 Binary ↔ Decimal 转换、Bytes、编码格式 3个核心概念
💡

这节课在 Python 里是干什么用的？

这是整个课程的"地基"。计算机本质上只认识 0 和 1，所有你看到的文字、图片、音乐，底层都是一串串二进制数。理解这个，你才能真正明白为什么 Python 里有那么多不同的数据类型（int、float、str……）——因为同一串 0 和 1，用不同的方式"解读"，就变成了不同的东西。这节课考试不会让你大量计算，但是选择题和概念题经常出现。

Binary（二进制） ↔ Decimal（十进制） 互转 📝 记住方法

Binary → Decimal（从右到左，每位乘以 2 的幂次）：

# 10101100 转十进制： # 位置： 7 6 5 4 3 2 1 0 ← 从右边数，从 0 开始 # 数字： 1 0 1 0 1 1 0 0 # # 计算：1×2⁷ + 0×2⁶ + 1×2⁵ + 0×2⁴ + 1×2³ + 1×2² + 0×2¹ + 0×2⁰ # = 128 + 0 + 32 + 0 + 8 + 4 + 0 + 0 # = 172 # 所以 10101100₂ = 172₁₀

Decimal → Binary（不断除以 2，收集余数，从下往上读）：

# 把 14 转成二进制： # 14 ÷ 2 = 7 余 0 ← 最低位 # 7 ÷ 2 = 3 余 1 # 3 ÷ 2 = 1 余 1 # 1 ÷ 2 = 0 余 1 ← 最高位 # # 余数从下往上读：1 1 1 0 # 所以 14₁₀ = 1110₂
💡 生活类比 十进制就是我们数数用的系统（0-9），满 10 进一位。二进制只有 0 和 1，满 2 就进一位——就像一个只有"开"和"关"两种状态的电灯开关，一排开关组合起来就能表示任何数字。

Bit、Byte 和编码格式 📝 记住概念
Bit（位） 一个 0 或 1，是计算机信息的最小单位
Byte（字节） 8 个 bit 组合在一起。
1 Byte = 8 Bits

同一串二进制数字，用不同方式"解读"，就是完全不同的东西：

# 二进制 1001000 可以被解读为： # 数字 72 # 字符 'H'（ASCII 编码） # 图片里的某种颜色（RGB 编码） # …… # 所以需要有统一的格式规范来告诉计算机"这串数字是什么意思"： # .mp3 / .wav → 音频 # .jpeg / .png → 图片 # ASCII / Unicode → 文本（Unicode 包含中文！）

这节课在 Python 里是干什么用的？

这节课是"编程思维"的起点。在动手写代码之前，你得先学会把问题分解成一步步清晰的指令——这就是算法（algorithm）的思维方式。同时，你得知道 Python 是什么类型的语言、如何运行，以及最基础的输出工具 print()。这一课的东西不难，但是"理解编程本质"的关键。

Algorithm vs Program 📝 记住区别
Algorithm（算法） 解决问题的步骤设计，和具体编程语言无关。可以用自然语言、伪代码（pseudo-code）或流程图（flowchart）来表达。
Program（程序） 用某种具体编程语言实现的算法。同一个算法可以用 Python、Java、C++ 等不同语言来实现。
💡 生活类比 Algorithm 是食谱（用文字写的做菜步骤），Program 是你真正下厨做出来的那道菜。食谱可以翻译成任何语言，但最终做菜时你必须用某种厨具（编程语言）来实现它。
Low-Level vs High-Level 语言 📝 记住分类
Low-Level（低级语言）High-Level（高级语言）
例子Machine code、AssemblyPython、Java、C++
可读性很差，接近 0/1 指令接近人类语言，可读性强
可移植性与特定机器绑定可以在不同机器上运行
翻译方式直接运行需要 Compiler 或 Interpreter 翻译

Python 是高级语言，用的是 Interpreter（解释器）——一行一行翻译执行，边翻边跑；C++ 用的是 Compiler（编译器）——先全部翻译成机器码，再统一跑。这就是为什么 Python 报错时你能看到精确的行号：解释器是一行行跑的，跑到哪行崩了就告诉你哪行。

print()、\n、\t 📝 记住用法 ⭐ 必掌握
# 基本用法 print("hello", "world") # hello world（默认 sep=" "） print("hello", "world", sep="-") # hello-world print("hello", end="") # 不换行，紧接着下一个 print # 打印变量 x = 7 print("The value of x is", x) # The value of x is 7 # 特殊字符 print("Line 1\nLine 2") # \n = 换行（new line） print("col1\tcol2") # \t = Tab 对齐（补空格到4的倍数） # 注释：# 开头的内容，Python 完全忽略 # 这是一行注释 x = 5 # 这是行末注释
LESSON 3 · Week 2 Variables、Type、input()、数学运算、Math 模块、Expression vs Statement 5个核心概念
🔢

这节课在 Python 里是干什么用的？

变量（variable）是编程的核心——它是你存数据的容器。这节课教你 Python 里有哪些基本数据类型，怎么在它们之间转换（type cast），怎么拿到用户的输入，以及怎么做数学计算。这些东西在之后的每一节课里都会用到，是最基础最核心的一课。

Variables（变量）和 Type（类型） 📝 记住类型 ⭐ 必掌握
# 赋值语句：variable_name = value number = 67 name = "Alice" pi = 3.14 is_student = True # 查看类型 print(type(12.3)) # <class 'float'> print(type("hello")) # <class 'str'> print(type(5)) # <class 'int'> print(type(True)) # <class 'bool'>

Type Cast（类型转换）：

int(3.14) # → 3 （直接截断小数部分，不四舍五入！） float(5) # → 5.0 str(42) # → "42" int("hello") # ❌ ValueError，文字不能转成数字！ int("42") # ✅ → 42 （纯数字字符串可以转）
⚠️ int() 截断不是四舍五入！ int(3.9) 结果是 3，不是 4。它直接把小数部分扔掉。要四舍五入用 round(3.9)。
input() 函数 ⭐ 必掌握 📝 记住坑

input() 让用户在键盘上输入内容，但它永远返回 string（字符串），就算你输入了数字也是字符串！

# ❌ 危险写法：忘记转类型 x = input("Enter a number: ") print(x + 1) # TypeError！str + int 不行 # ✅ 正确写法：记得 type cast x = int(input("Enter an integer: ")) print(x + 1) # 现在没问题了 y = float(input("Enter a decimal: ")) print(y * 2) # 也没问题
💡 生活类比 input() 就像一个总是用中文回答的翻译官——不管你用什么语言问他，他的回答永远是字符串格式。你要的是数字，就要自己"翻译"一下（type cast）。
数学运算 + 优先级 📝 记住运算符 ⭐ 必掌握
10 + 3 # → 13 加法 10 - 3 # → 7 减法 10 * 3 # → 30 乘法 10 / 3 # → 3.333... 除法（结果总是 float！） 10 ** 3 # → 1000 幂次（exponentiation，指数） 10 // 3 # → 3 整除（floor division，向下取整） 10 % 3 # → 1 取余（modulus，余数） # 运算优先级（从高到低）： # 1. () 括号最高 # 2. ** 幂次 # 3. * / % // 乘除类 # 4. + - 加减类
⚠️ 最容易搞混的三个运算符 / 普通除法，结果永远是 float（比如 6/2 = 3.0）。
// 整除，结果向下取整（比如 7//2 = 3，-7//2 = -4！）。
% 取余（比如 7%3 = 1），可以用来判断奇偶：n%2 == 0 说明 n 是偶数。

常用内置函数 + Math 模块：

# 内置函数（直接用，不需要 import） abs(-5) # → 5 绝对值 pow(2, 10) # → 1024 等同于 2**10 round(7.948, 2) # → 7.95 保留 2 位小数 # math 模块（需要先 import） import math math.sin(math.pi / 4) # sin(45°) math.cos(math.pi) # cos(180°) = -1.0 math.pi # → 3.141592653589793
Expression（表达式）vs Statement（语句） 🧠 理解概念
Expression（表达式） 能被计算出一个值的代码片段。
例：1/2、a + 5、"hello" + "world"
Statement（语句） 执行某个动作的完整指令，不一定有返回值。
例：x = 5（赋值语句）、print("hi")

程序从上到下一行一行执行 statement，遇到 expression 就计算出值再继续。

LESSON 4 · Week 3 Boolean、关系运算符、and/or/not、if/elif/else 条件语句 4个核心概念
🚦

这节课在 Python 里是干什么用的？

到目前为止，程序都是从第一行跑到最后一行，没有任何"判断"。这节课教你如何让程序根据条件走不同的路——这才是让程序"聪明"起来的关键。Boolean（布尔）值是做判断的基础，if/elif/else 是控制程序走向的门。几乎所有有意义的程序都需要条件判断。

Boolean 值和关系运算符 📝 记住运算符 ⭐ 必掌握
# 关系运算符：比较两个值，结果是 True 或 False 7 == 7 # True （等于） 7 != 5 # True （不等于） 7 > 5 # True （大于） 7 < 5 # False （小于） 7 >= 7 # True （大于等于） 7 <= 6 # False （小于等于）
⚠️ == 和 = 绝对不能搞混！ = 是赋值（assignment）：x = 5 把 5 存进 x。
== 是比较（comparison）：x == 5 问"x 等于 5 吗？"结果是 True 或 False。
这是最经典的 bug 来源之一，写 if x = 5: 会直接 SyntaxError。

浮点数比较的坑：

x = 1.1 + 2.2 print(x == 3.3) # False！！浮点数有精度误差 # 正确做法：检查是否"足够接近" epsilon = 0.0001 print(abs(x - 3.3) < epsilon) # True ← 这才是正确比较浮点数的方式
Boolean 运算符：not、and、or 🧠 理解逻辑 📝 记住真值表
# not（取反）：True ↔ False not True # False not False # True # and（与）：两边都 True 才是 True True and True # True True and False # False False and True # False # or（或）：至少有一边是 True 就是 True True or False # True False or False # False # 优先级：not > and > or（和数学里 × > + 一样，先算 not） not True or False # = (not True) or False = False or False = False
💡 生活类比 and：你要"又有学生证，又带了作业"才能进实验室——两个条件都得满足。
or：你"有现金或者有手机支付"就能买奶茶——满足其中一个就行。
not：把 True/False 翻转，就像"不是学生"。
⚠️ Short-circuit evaluation（短路求值） True or (3 + "error") 结果是 True——因为左边已经是 True，or 就不看右边了。右边那个会崩溃的表达式根本没有被执行。and 同理：False and anything 直接就是 False。
if / elif / else 条件语句 ⭐ 必掌握 🧠 理解执行流程
# 基本结构 if condition1: # condition1 为 True 时执行 print("condition 1 is True") elif condition2: # condition1 为 False，且 condition2 为 True 时执行 print("condition 2 is True") else: # 以上全部为 False 时执行（else 不是必须的） print("all conditions are False") # 嵌套 if（nested conditionals） if x == y: print('x and y are equal') else: if x < y: print('x is less than y') else: print('x is greater than y')

重点规则：

  • 一旦某个 if 或 elif 的条件为 True，执行完那个块后，剩下的 elif 和 else 全部跳过
  • elif 可以有任意多个
  • else 不是必须的，可以没有
  • 缩进（indentation）是关键——Python 靠缩进判断哪些代码属于哪个块
LESSON 5 · Week 4 Functions：def、参数、Scope、Void/Fruitful、Docstring、常见误区 6个核心概念
⚙️

这节课在 Python 里是干什么用的？

Function（函数）是编程里最重要的抽象工具。有了函数，你就能把一段重复的代码封装起来，给它起个名字，之后随时调用——不需要一遍遍复制粘贴。这节课讲的 scope（作用域）、return vs print、docstring 这几个概念，是 COMP202 里考试和作业最高频的考点之一，务必搞清楚。

定义函数（def）和调用函数 ⭐ 必掌握 📝 记住语法
# 定义函数（定义 ≠ 执行！） def greet(name): print("Hello, " + name) print("Welcome to McGill!") # 调用函数（定义之后才能调用） greet("Alice") # Hello, Alice \n Welcome to McGill! greet("Bob") # Hello, Bob \n Welcome to McGill! # 返回多个值（实际上是返回一个 tuple） def calculator(a, b): add = a + b subtract = abs(a - b) multiply = a * b divide = a / b return add, subtract, multiply, divide result = calculator(10, 5) print(result) # (15, 5, 50, 2.0)
⚠️ 定义函数不等于运行！ 写了 def greet(): 只是告诉 Python "我定义了这个函数"，函数体里的代码不会立刻执行。必须要调用 greet() 才会真正运行。这是初学者最常犯的误解。
Scope（作用域）：Local vs Global 变量 🧠 理解逻辑 ⭐ 必掌握
Local Variable（局部变量） 在函数体内部定义的变量（包括参数）。只在函数内部"存在"，函数结束后消失。
Global Variable（全局变量） 在所有函数外部定义的变量。函数内部可以读取，但不能直接修改（修改需要 global 关键字）。
def my_function(): x = 5 # x 是 local variable，只在函数里存在 print("inside:", x) X = 10 # X 是 global variable my_function() # inside: 5 print("outside:", X) # outside: 10 ← 函数里的 x 和外面的 X 是两个独立的变量 # ❌ 这会报错：在函数内试图修改全局变量 count = 5 def bad_function(): count += 7 # UnboundLocalError！Python 认为你在访问一个"还没被赋值的局部变量"
💡 生活类比 函数就像一个独立的工作间。工作间里的工具（local variable）只在里面用，出来就不见了；外面走廊的公共工具（global variable）你可以在工作间里看到和用，但你不能在工作间里把公共工具扔掉或换掉（除非你特别申请——用 global 关键字）。
Void vs Fruitful 函数：print 和 return 的区别 🧠 理解逻辑 ⭐ 超高频考点
Void Function（无返回值函数） 没有 return 语句（或只有 return 没有值）。实际上返回 None。
用 print() 显示内容。
Fruitful Function（有返回值函数） 有 return 某个值。调用者可以把这个值存进变量或继续用。
不用 print，而是 return。
# ❌ 常见误区：print 不等于 return def bad_add(a, b): print(a + b) # 只是显示，不返回值 result = bad_add(3, 4) print(result) # None ← 什么都没返回！ # ✅ 正确做法：用 return def good_add(a, b): return a + b # 返回计算结果 result = good_add(3, 4) print(result) # 7 ← 这才对 # return 立刻结束函数，后面的代码不会执行 def early_exit(x): if x < 0: return 0 # 直接退出，下面的不会跑 return x * 2
⚠️ 记得接住 return 值！ test() 调用函数但没把返回值存起来，结果就直接消失了。正确写法是 value = test()，把返回值存进变量，之后才能用。
Docstring（文档字符串）的格式 📝 记住格式 ⭐ 必掌握

Docstring 写在函数定义后的第一行，用三引号包裹，解释函数的用途。这是 COMP202 作业的标准要求，格式不对会扣分！

def area_circle(radius): """ (num) -> num Returns the area of a circle with a given radius. >>> area_circle(20) 1256.64 >>> area_circle(0) 0 """ import math return math.pi * pow(radius, 2.0)

Docstring 三个组成部分：

  • Type contract（类型契约）：第一行，格式 (input types) -> return type
  • Description（描述）：函数做什么事
  • Examples（例子）：用 >>> 开头，写输入和预期输出（这就是 L13 的 doctest！）
LESSON 6 · Week 5 While Loop：语法、更新条件、无限循环、Shorthand Assignment 4个核心概念
🔄

这节课在 Python 里是干什么用的？

While loop 是让程序"重复做某件事"的工具。区别于 for loop（L7 会讲，用于遍历已知序列），while loop 适合"不知道要重复几次，但知道什么时候停"的场景——比如"一直让用户输入，直到输入正确为止"，或者"一直计算，直到满足某个条件"。While loop 配合 L19 的 try/except 一起用，是处理用户输入错误的经典写法。

While Loop 语法 + 执行流程 🧠 理解逻辑 ⭐ 必掌握
# 基本语法 while condition: # 每次循环开始前都会检查这个条件 # 条件为 True 时执行循环体 # ⚠️ 循环体里必须有代码让条件最终变成 False！ # 完整例子：倒数 n = 3 while n > 0: print('Value of n is:', n) n = n - 1 # ← 关键：更新 n，让条件最终变 False # 输出：3、2、1（n=0 时 0>0 为 False，停止）
💡 生活类比 While loop 就像洗衣机的程序："只要衣服还没洗干净，就继续洗"。每洗一遍，检查一下衣服干不干净（条件）——如果还不行，继续洗（循环）；干净了就停下来（条件变 False，退出循环）。关键是：洗衣过程必须让"干净程度"越来越高，否则就会永远洗不完（无限循环）。
无限循环（Infinite Loop）：三种常见原因 🧠 理解原因 ⭐ 必须避免
# ❌ 原因 1：根本没有更新条件 n = 3 while n > 0: print('hello') # n 永远是 3，循环永远不停 # ❌ 原因 2：更新方向错了 n = 6 while n > 5: n = n + 1 # n 越来越大，永远大于 5 # ❌ 原因 3：每次都把条件重置成常量 n = 6 while n > 5: n = 7 # n 永远被重置回 7，永远大于 5

在 Thonny 里如果陷入无限循环，按 Ctrl + C 强制停止程序。

Shorthand Assignment（复合赋值运算符） 📝 记住写法

这些是常用的简写，在 while loop 里更新计数器时特别常用：

x += 3 # 等价于 x = x + 3 x -= 3 # 等价于 x = x - 3 x *= 3 # 等价于 x = x * 3 x /= 3 # 等价于 x = x / 3 x **= 2 # 等价于 x = x ** 2 x %= 3 # 等价于 x = x % 3 x //= 3 # 等价于 x = x // 3 # 实际使用：在 while loop 里更新计数器 count = 0 while count < 5: print(count) count += 1 # 比写 count = count + 1 更简洁

这节课的核心是"让程序重复干活"。For loop 让你能批量处理数据；recursion（递归）给你另一种解决问题的思路；variable scope（变量作用域）教你搞清楚变量什么时候"存在"、什么时候"消失"。这三件事是写出真正有用程序的基础。

Recursion（递归） 🧠 要理解逻辑 ⭐ 必掌握

Recursion 就是一个 function（函数）调用自己。写 recursive function（递归函数）必须搞定两件事，缺一不可：

  • Base case（基础情况/终止条件）：告诉程序什么时候该停下来，不然会无限循环崩溃
  • Recursive step（递归步骤）：调用自己，但每次都要往 base case 靠近一步
def countdown(n): if n <= 0: # ← Base case，到这里就停！ print('Blastoff!') else: print(n) countdown(n - 1) # ← Recursive step，n 每次减 1，往 base case 靠
💡 生活类比 想象你在数一叠盘子有多高。你的做法是：拿起最上面那个，数"1"，然后让旁边的人去数剩下的——旁边那个人也用同样的办法。一直到最后一个人发现面前没有盘子了（base case），就报"0"，然后答案一层层往上传回来。这就是 recursion 的运作方式。

Recursion vs 普通 loop，什么时候用哪个？

  • 递归次数很多、想省内存、追求速度 → 用 loop（while 或 for）
  • 不那么 performance-sensitive（对性能要求不高），但想让代码更简洁 → 用 recursion

Variable Scope（变量作用域） 🧠 要理解逻辑

Python 里最小的 "local（局部）" 范围是 function body（函数体），不是 if、for、while 的 block（代码块）！

这意味着：在 if 或 for 里定义的变量，只要那段代码跑过了，在外面照样能用。但如果那段代码压根没执行，变量就不存在，访问它会报 NameError。

x = 5 if x > 0: y = x print(y) # ✅ 没问题！if 跑过了，y 存在，输出 5 # ───────────────────────────────────── x = 5 if x > 0: y = x else: z = -x # x=5 > 0，所以 else 没执行 print(z) # ❌ NameError！z 从来没被创建过
💡 生活类比 就像只有你真正打开过一个包裹，里面的东西才到你手上。如果那个 if/else 分支从没执行，变量就从来没被"放进"内存——你去拿，当然拿不到。

这节是 Python 的"底层逻辑"课。理解 reference（引用）和 copy（拷贝）能避免一类极其隐蔽的 bug：你以为自己改了"副本"，实际上改的是"原件"，然后原来的数据莫名其妙就变了。这在 assignment 和 interview 里都非常常考。

Object Identity（对象标识） — is vs == 🧠 理解逻辑

Python 里每个对象都有一个唯一的内存地址，用 id() 可以查看。

  • ==：比较两个变量的值是否相同
  • is：比较两个变量是否指向内存里同一个对象（相当于 id(a) == id(b)）
a = "hello" b = "hel" b += "lo" a == b # True ← 值相同 a is b # False ← 不是同一个对象（不同 memory address） # b = a 是复制引用，所以 is 为 True a = 400 b = a # b 和 a 指向同一块内存 a is b # True b = 200 # b 现在指向一个新对象 a is b # False
💡 生活类比 == 是比较两个人的名字是不是一样；is 是比较他们是不是同一个人（同一张身份证号）。两个不同的人都叫"张三"（== True），但他们是两个人（is False）。
Aliasing、Shallow Copy、Deep Copy 三者的区别 🧠 理解逻辑 ⭐ 重难点

这三个概念经常放在一起，但本质上是完全不同的三件事。区分它们最好的方式是分 1D list 和 2D list（nested list） 两种情况来看。

① Aliasing（别名）——根本不是 copy！

list2 = list1 不会创建新 list，只是让 list2 这个变量名也指向同一块内存。两个名字，同一个东西。

list1 = [1, 2, 3] list2 = list1 # ← Aliasing！list2 不是新 list，只是同一个 list 的另一个名字 list1[0] = 99 print(list1) # [99, 2, 3] print(list2) # [99, 2, 3] ← list2 也变了！因为它们本来就是同一个东西 print(list1 is list2) # True ← id 完全相同，确认是同一个对象
💡 类比 Aliasing 就像给同一个人起了两个外号——"老张"和"张老师"指的是同一个人，跟"老张"说"剪头发"，"张老师"的头发也变短了。

② 1D list 的情况：用 .copy() 或 [:] 就足够了

对于普通的一维 list（元素都是 int、str 等 immutable 值），.copy() 和 [:] 会创建一个新的 list 对象，两个 list 从此独立，互不影响。

list1 = [1, 2, 3] list2 = list1.copy() # 方式 1：.copy() list3 = list1[:] # 方式 2：切片，效果完全一样 print(list1 is list2) # False ← 是两个不同的 list 对象了 list1[0] = 99 print(list1) # [99, 2, 3] print(list2) # [1, 2, 3] ← 不受影响 ✅ print(list3) # [1, 2, 3] ← 也不受影响 ✅

③ 2D list（nested list）的情况：.copy() 不够，必须用 copy.deepcopy()

这是最容易出 bug 的地方！当 list 里面还套着 list 时，.copy() 只复制了外层结构，内层的子 list 仍然是 shared（共享）的 reference——改内层，两边都会变。

import copy original = [[1, 2], [3, 4]] # ⚠️ .copy() 在 2D list 上只是 shallow copy——外层新建，内层仍然共享 shallow = original.copy() print(original is shallow) # False ← 外层 list 确实是新的 print(original[0] is shallow[0]) # True ← 但内层 [1,2] 还是同一个！ original[0][0] = 99 # 修改内层子 list print(original) # [[99, 2], [3, 4]] print(shallow) # [[99, 2], [3, 4]] ← shallow 也变了！😱 # ✅ 正确做法：copy.deepcopy() 把每一层都真正复制 deep = copy.deepcopy(original) print(original[0] is deep[0]) # False ← 内层子 list 也是全新的 original[0][0] = 0 print(original) # [[0, 2], [3, 4]] print(deep) # [[99, 2], [3, 4]] ← deep 完全不受影响 ✅
💡 类比（用公寓来理解三者） Aliasing：你把你家的地址给了朋友，他们来"住"其实是住你的房子，搬走任何东西你都感受得到。

Shallow copy（1D够用，2D不够）：你搬进一套"一模一样的新公寓"，但公寓里的家具（内层子 list）还是租来的公共家具，朋友在他那边把公共沙发拆了，你这边也发现沙发没了。

Deep copy：你搬进一套新公寓，连里面所有家具都重新买了一套全新的。朋友怎么折腾他那边都和你无关。
Aliasing b = aShallow Copy .copy() / [:]Deep Copy copy.deepcopy()
创建新 list？❌ 同一个对象✅ 外层新建✅ 每层都新建
1D list 改外层元素两边都变互不影响 ✅互不影响 ✅
2D list 改内层元素两边都变两边都变 ⚠️互不影响 ✅

⚠️ 结论：记住这个判断流程 你的 list 是 1D（里面都是数字、字符串）→ 用 .copy() 或 [:]，足够了。
你的 list 是 2D 或更深的嵌套结构 → 必须用 copy.deepcopy()，否则内层还是共享的。
Mutable vs Immutable 📝 记住分类
✅ Mutable（可变） List、Dictionary（字典）
可以原地修改内容，id 不变
❌ Immutable（不可变） String、int、float、bool
"修改"等于创建新对象，id 会变
# String — immutable，改不了 t = "hello" t[0] = "1" # ❌ TypeError，字符串不能这样改 # List — mutable，随便改 lst = [1, 2, 3] lst[0] = 99 # ✅ 完全没问题

一个重要的 implication（含义）：把 list 传给 function（函数）时，function 拿到的是 reference（引用）。如果 function 在里面修改了 list，外面的原始 list 也会变！这和 int、string 完全不同，注意不要踩坑。

Doctest（文档测试） 📝 记格式 ⭐ 必掌握

想象你开了一家奶茶店，每次上新品都要自己喝一口确认味道对不对。Doctest 就是这个"自动试喝"系统——你在写函数的时候，顺手在 docstring 里写几个"如果我传这个进去，应该出来什么"的例子，Python 就会帮你自动验证这些例子对不对。

怎么用 doctest？两行搞定：

import doctest # 第一行：导入 doctest 模块 doctest.testmod() # 第二行：运行所有测试（放在文件最底部）

Doctest 怎么写？ 在 docstring 里，每一行测试以 >>> 开头，下一行写你期望的输出：

def add(x, y): """ (num, num) -> num Returns the sum of x and y >>> add(2, 2) 4 >>> add(-2, 3) 1 >>> add(1.2, 3.5) 4.7 """ summation = x + y # 正确写法 return summation import doctest doctest.testmod() # 全部通过 → 什么都不打印（安静 = 好事！）
💡 生活类比 >>> add(2, 2) 就是你在说"好，我来测试：往机器里放 2 和 2，机器应该吐出 4"。
如果你写的函数里 x * y 写错成乘法，机器吐出了 4（凑巧对了），但 add(-2, 3) 就会暴露 bug——应该是 1，但乘法会给你 -6。所以好的 doctest 要覆盖边界情况（edge cases），不能只测"正常情况"。
⚠️ 重点细节：doctest 对格式极其敏感 doctest 是逐字符比较的！如果你的函数返回 4.699999999999999（浮点数精度问题），但你 docstring 写的是 4.7，测试会失败。解决方法：用 round() 处理浮点数结果，或者把期望值也写成精确的格式。

如果函数会 raise exception（抛出异常），doctest 怎么写？

格式固定，必须背下来：先写 Traceback (most recent call last):，中间的 stack trace（调用栈）可以省略，最后一行写 exception 的名字和消息：

def print_multiples(n, times): """ >>> print_multiples('2', 4) Traceback (most recent call last): ... TypeError: the function expects two integers as input """ if not isinstance(n, int) or not isinstance(times, int): raise TypeError("the function expects two integers as input") for i in range(1, times + 1): print(n * i)
doctest 要点说明
测试行以 >>> 开头Python 靠这个识别哪行是测试
期望输出在下一行（不加 >>>）如果函数返回 None（无返回值），不写期望输出
异常测试：先写 Traceback 头，再写 exception 类型中间 stack trace 用 ... 省略
全部通过 = 没有输出有输出才说明有 test 挂掉了
Debugging（调试）：三种 Error 类型 📝 记住分类 🧠 要理解逻辑

Python 里的 bug 分三大类，记住这个分类非常重要——因为它决定了你用什么方式去找 bug：

① Stylistic Errors
（代码风格问题） 不影响运行结果，但让代码很难读。比如变量命名乱七八糟、缩进不统一。
不是"真"的 error，但会被扣分！
② Syntax Errors
（语法错误） 代码语法写错了，Python 连运行都不运行，直接报错。
最先被检测到，修起来也相对好找。
③ Runtime Errors
（运行时错误） 语法没问题，但执行到某一行的时候崩了。比如除以零、类型不对。
发现起来比语法错误晚一步。
④ Logic Errors
（逻辑错误） 程序跑完了，但结果不对。Python 完全不报错，是最难找的！
完全靠你自己的大脑去发现。

每种 error 的例子：

# ❌ Syntax Error — 忘记写冒号 x = int(input("Enter a number: ")) if x > 100 # 少了冒号 ':'，Python 直接报 SyntaxError print("You think big!") # ❌ Runtime Error — 类型不对 x = input("Enter a number: ") # input() 返回的是 str（字符串）！ if x > 100: # ← TypeError: '>' not supported between str and int print("You think big!") # ❌ Logic Error — 代码跑通了，但结果不对 x = input("Enter your favourite number: ") print("Multiplied by 2:", x * 2) # 输入 5 → 输出 "55"，因为 "5" * 2 = "55"（字符串重复，不是数学乘法！）
💡 生活类比 Syntax Error 就像你写了一封语法错误的申请信，秘书直接退回来，都没有递进去。
Runtime Error 就像申请信格式正确，但进了系统发现你填的身份证号少一位，系统崩了。
Logic Error 就像申请顺利通过，但你把目的地填错了，最后飞到了错误的城市，没有任何人提醒你。

怎么解决这三类 bug？

  • Syntax / Runtime Errors：认真读报错信息，它会告诉你第几行出错了、出了什么问题。配合 Google 或课件查。
  • Logic Errors（最难！）：
    • 在可疑的地方加 print()，把中间变量的值打印出来，追踪哪一步开始不对
    • 用 IDE 的 debug 工具（debugger）打断点（breakpoint），一步一步执行看变量变化
    • Rubber ducking（橡皮鸭调试法）：对着橡皮鸭（或者任何人、任何东西）大声把代码逻辑讲一遍——很神奇，讲着讲着你自己就发现问题了
⚠️ 最容易踩的坑：Syntax Error 会屏蔽其他所有错误 如果你的代码里有 syntax error，Python 根本不会运行，所以 runtime error 和 logic error 都不会被发现。永远先修 syntax error，修完再跑，再看还有没有其他问题。
Raising Exceptions（主动抛出异常） 🧠 理解逻辑 📝 记住语法 ⭐ 必掌握

Exception（异常）是程序在运行时遇到"说不通"的事情时发出的警报。Python 自己会 raise 一些 exception（比如 TypeError、ZeroDivisionError）。但作为程序员，你也可以——而且应该——主动 raise exception，来告诉别人"你传进来的东西不对"。

💡 生活类比 你开了一家外卖店，有人点了"0份炒饭"。机器不会报错，但这明显不合理。作为厨师，你应该主动说"不行，至少要点1份"然后拒绝这个订单，而不是默默地做了0份然后送出去一个空盒子。这就是 raise Exception 的意义——提前把问题截住，让 bug 更快暴露、更容易找。

语法：

raise SomeExceptionType("错误信息写在这里") # 常用的 exception 类型： raise TypeError("message") # 类型不对时用这个 raise ValueError("message") # 类型对但值不合理时用这个

完整例子：

def divide(a, b): """ (int, int) -> float Divides a by b. >>> divide(10, 2) 5.0 >>> divide(10, 0) Traceback (most recent call last): ... ValueError: The divisor 'b' cannot be zero. >>> divide(10.0, 2) Traceback (most recent call last): ... TypeError: Both arguments must be integers. """ if not isinstance(a, int) or not isinstance(b, int): raise TypeError("Both arguments must be integers.") if b == 0: raise ValueError("The divisor 'b' cannot be zero.") return a / b import doctest doctest.testmod()
Exception 类型什么时候 raise例子
TypeError传进来的类型不对函数要 int，你传了 str
ValueError类型对，但值不合理函数要正数，你传了 -5
ZeroDivisionError除数是零通常 Python 自己 raise，但你也可以提前截住
⚠️ TypeError vs ValueError 别搞混 类比一下：你订了一杯"珍珠奶茶"，结果店员给你端来了一碗炒饭 —— 这是 TypeError（类型完全不对）。但如果你订了"微糖"，结果给你超甜的 —— 这是 ValueError（类型没错，就是个饮料，但值不是你要的）。

Exception 和 Doctest 的关系： 当你的函数里有 raise，你必须在 doctest 里写对应的 exception 测试案例，格式固定（见上面 doctest 章节）。这是 COMP202 作业里常见的失分点——写了 raise 但忘了在 docstring 里加对应测试。

LESSON 14 · Week 10 Dictionary（字典）：创建、访问、增删、方法、遍历 6个核心概念
📖

这节课在 Python 里是干什么用的？

List 让你存一排东西，但每次找东西都要靠位置（index）。Dictionary 则是给每个东西贴上一个"名字标签"（key），你直接喊标签名字就能拿到对应的东西，不用知道它排在第几位。这是从"书架"升级到"图书馆检索系统"——数据量一大，dict 的查找速度远比 list 快。几乎所有实际项目都大量用 dict。

Dictionary 的创建 + Key/Value 规则 📝 记住规则 ⭐ 必掌握

Dictionary 的每个元素叫做 item（条目），由一个 key（键） 和一个 value（值） 组成，中间用冒号 : 分隔。

# 方式 1：直接用花括号 {} heights = {'Sasha': 182, 'Rohan': 168, 'Kate': 177} # 方式 2：用 dict() 函数创建空字典 empty_dict = dict() also_empty = {} # 两种方式都可以
✅ Key 的规则 必须是 immutable（不可变）类型：str、int、float、bool、tuple。
在同一个 dict 里必须唯一（不能重复）。
✅ Value 的规则 可以是任意类型，包括 list、dict、None。
不需要唯一，可以重复。
⚠️ List 不能当 Key！ 因为 list 是 mutable（可变的），所以不能作为 dictionary 的 key，否则会报 TypeError: unhashable type: 'list'。但 tuple 可以！
访问、添加、删除 📝 记住语法
heights = {'Sasha': 182, 'Rohan': 168, 'Kate': 177} # 访问 value：dict[key] print(heights['Rohan']) # 168 # 添加 / 修改：直接赋值就行 heights['Rohan'] = 175 # key 已存在 → 覆盖旧值 heights['Lena'] = 165 # key 不存在 → 新增一条 # 删除：del dict[key] del heights['Sasha'] print(heights) # {'Rohan': 175, 'Kate': 177, 'Lena': 165}
⚠️ KeyError：访问不存在的 key 会崩！ heights['NotExist'] 会直接抛出 KeyError。安全写法是用 dict.get(key)——找不到时返回 None 而不是崩溃。
常用方法速查 📝 要背住
方法 / 函数作用注意事项
d.get(key)安全取值，找不到返回 None不报 KeyError，推荐用这个代替直接 d[key]
d.pop(key)删除并返回对应 valuekey 不存在会报 KeyError

遍历 Dictionary 🧠 理解逻辑
scores = {'Alice': 90, 'Bob': 85, 'Carol': 92} # 方式 1：直接 for loop → 遍历所有 key for key in scores: print(key, scores[key]) # 方式 2：用 .items() 同时拿 key 和 value（更 Pythonic！） for name, score in scores.items(): print(name, "scored", score)
💡 生活类比 遍历 dict 就像翻通讯录——方式 1 是一页一页翻看名字（key），然后手动去查电话；方式 2（用 .items()）是直接看每一行的"姓名 + 电话"，一次拿到两个信息，省事多了。
经典用法：用 Dict 统计出现次数 🧠 理解逻辑 ⭐ 必掌握

这是 COMP202 里 dictionary 最常考的实际应用！核心思路：key 是"东西是什么"，value 是"出现了多少次"。

def count_occurrences(input_list): """ (list) -> dict Returns a dict counting how many times each element appears. >>> count_occurrences(['a', 'b', 'a', 'c', 'b', 'a']) {'a': 3, 'b': 2, 'c': 1} """ count = {} for element in input_list: if element in count: count[element] += 1 # 已经见过，计数 +1 else: count[element] = 1 # 第一次见，初始化为 1 return count
💡 生活类比 你在统计班里有多少人喜欢不同口味的奶茶。拿着一张白纸，每碰到一个同学：先找白纸上有没有他的口味；有 → 在旁边的数字上 +1；没有 → 新写一行，数字从 1 开始。这就是这段代码的逻辑，非常直白。
LESSON 15 · Week 10 File I/O（文件读写）、Pickle、CSV 3个核心概念

这节课在 Python 里是干什么用的？

到目前为止，你写的所有程序关了就什么都没了——变量全消失。File I/O（Input/Output，输入/输出）让你的程序能和硬盘上的文件交互：把数据存下来，或者读取别人给你的数据。这是写出"真正有用的程序"的关键一步——任何需要处理数据文件的应用都靠这个。

文件操作三步走：Open → Read/Write → Close 📝 记住流程 ⭐ 必掌握
💡 生活类比 操作文件就像借图书馆的书：先去借书（open），然后看书或写笔记（read/write），最后一定要还书（close）——不还书（不 close）会导致文件被锁住，其他程序没法用，还可能丢数据。

open() 的 mode（模式）参数：

mode含义注意
'r'read（只读）文件不存在会报错
'w'write（写入/覆盖）⚠️ 文件已存在会清空重写！
'a'append（追加）在文件末尾添加，不会清空原内容
'rb' / 'wb'二进制读/写Pickle 专用，处理非文本数据

读取文件（Reading）：

# 方式 1：用 for loop 逐行读（最常用！内存友好） fobj = open('quotes.txt', 'r') for line in fobj: print(line) # 注意：每行末尾会有 '\n'（换行符） fobj.close() # 方式 2：一次性读全部内容成一个 string fobj = open('quotes.txt', 'r') content = fobj.read() fobj.close() # 方式 3：读成一个 list，每行是一个元素 fobj = open('quotes.txt', 'r') lines = fobj.readlines() # → ['line1\n', 'line2\n', ...] fobj.close()

写入文件（Writing）：

# 覆盖写入（原内容会被清空！） fobj = open('output.txt', 'w') fobj.write('Hello, world!\n') # write() 不会自动换行，要手动加 '\n' fobj.write('Second line\n') fobj.close() # 追加写入（保留原内容） fobj = open('output.txt', 'a') fobj.write('This is appended.\n') fobj.close()
⚠️ 两个最容易忘的坑 1. 一定要 close()！没 close 可能导致数据没有真正写进文件（数据还在缓冲区）。
2. 'w' 模式会直接清空原文件，不可恢复。想保留原内容就用 'a'。
Pickle 模块 📝 记住语法

普通文件 I/O 只能存字符串（文本）。但如果你想把一个 list 或 dict 直接存进文件，不想手动转成字符串，就用 pickle！

💡 生活类比 普通文件 I/O 就像把东西拍成照片存档——别人也能看，但可能失真（原本是 list，变成了一串文字）。Pickle 则像把东西真空密封存档——原封不动，但只有你自己（Python）能打开，别人（Excel、记事本）打开看到的是乱码。
import pickle # 存（dump）：把 Python 对象写进文件 data = [1, 3, 1001, 'a', [7, 8]] f = open('data.pkl', 'wb') # 注意：模式是 'wb'（write binary） pickle.dump(data, f) f.close() # 取（load）：从文件还原 Python 对象 f = open('data.pkl', 'rb') # 注意：模式是 'rb'（read binary） saved_data = pickle.load(f) f.close() print(saved_data) # [1, 3, 1001, 'a', [7, 8]] ← 完全还原！
普通文件 I/O（.txt）Pickle（.pkl）
存储格式纯文本，人能读二进制，只有 Python 能读
能存的类型只能存字符串任意 Python 对象
适合场景和其他人/程序共享数据你自己的程序内部使用
CSV 文件 📝 了解概念

CSV 是一种用逗号分隔数据的纯文本格式，Excel / Numbers 可以直接打开。对 Python 来说，读写 CSV 和读写普通 .txt 文件方法完全一样——只是文件里的内容格式是"用逗号隔开的"。

# CSV 文件长这样（文本内容）： # Movie1,80,100,90,100 # Movie2,90,93,100,100 # Python 读取 CSV：和读普通 txt 一样 fobj = open('movies.csv', 'r') for line in fobj: parts = line.strip().split(',') # strip() 去掉换行符，split 按逗号切 print(parts) # → ['Movie1', '80', '100', '90', '100'] fobj.close()
LESSON 16 · Week 11 Tuple（元组）、Packing/Unpacking、Enumerate 4个核心概念

这节课在 Python 里是干什么用的？

Tuple 是 list 的"只读版"——结构一样，但创建后就不能改了。这个"不能改"不是缺点，是特性：当你想保证一组数据不被意外修改时（比如坐标、日期），用 tuple 就对了。另外，enumerate() 是个超实用的小工具，能让 for loop 同时拿到"元素"和"位置"，是写遍历代码的一大利器。

Tuple 基础：创建、索引、不可修改 📝 记住特性 ⭐ 必掌握
# 创建 tuple：用圆括号 () coords = (45.5, -73.6) tup = (45, 23, 'abc') # 索引和 list 一样 print(tup[1]) # 23 print(tup[-1]) # 'abc' # tuple 转换：tuple() 把其他序列变成 tuple t = tuple('lupins') print(t) # ('l', 'u', 'p', 'i', 'n', 's') # ❌ 不能修改！ tup[0] = 99 # TypeError: 'tuple' object does not support item assignment
💡 生活类比 List 就像铅笔写的购物清单，随时可以划掉、添加。Tuple 就像刻在石头上的清单——就算你以后想改，也改不了。这在有些场景下反而是好事，比如你存了一个 GPS 坐标，你绝对不希望它被程序里某个 bug 偷偷改掉。

Packing（打包）& Unpacking（解包）& 多重赋值 🧠 理解逻辑 ⭐ 必掌握

Packing（打包）：把多个值"打包"成一个 tuple，括号可以省略：

my_tuple = 0, 3, 7 # 等价于 my_tuple = (0, 3, 7)

Unpacking（解包）：把 tuple / list / string 里的值一次性赋给多个变量：

my_tuple = (0, 3, 7) x, y, z = my_tuple print(x, y, z) # 0 3 7 # 多重赋值（multiple assignment）也是同一个道理 city, population, area = 'Montreal', 1704694, 431.5 print(city) # Montreal print(population) # 1704694
⚠️ 左右数量必须一致，否则 ValueError x, y = (1, 2, 3) 会报 ValueError: too many values to unpack (expected 2)。左边变量数量必须等于右边元素数量，不多不少。

For loop 与 list of tuples（这个考试很爱考！）：

favorite_languages = [('Bob', 'C++'), ('Rahul', 'Python'), ('Yifan', 'R')] # 方式 1：靠 index 取 for tup in favorite_languages: print(tup[0], tup[1]) # 方式 2：直接解包（更清晰，推荐！） for name, language in favorite_languages: print(name, language) # Bob C++ / Rahul Python / Yifan R

# list of tuples → dict（每个 tuple 是 (key, value) 对） list_of_pairs = [('Montreal', 1.78), ('Rome', 2.87), ('Tokyo', 9.27)] city_pop = dict(list_of_pairs) # {'Montreal': 1.78, 'Rome': 2.87, 'Tokyo': 9.27} # dict → list of tuples（用 .items()） items_view = city_pop.items() as_list = list(items_view) # [('Montreal', 1.78), ('Rome', 2.87), ('Tokyo', 9.27)]
enumerate()：遍历时同时拿到 index 和 value ⭐ 必掌握 📝 记住用法

以前你要在 loop 里同时知道"现在是第几个"，得自己设一个计数器变量。enumerate() 帮你自动做这件事：

s = "banana" # 没有 enumerate 的老写法（麻烦） i = 0 for char in s: print(char, "has index", i) i += 1 # 用 enumerate 的优雅写法 ✅ for index, char in enumerate(s): print(char, "has index", index) # b has index 0 / a has index 1 / n has index 2 ... # enumerate 也适用于 list、tuple、dict fruits = ['apple', 'banana', 'cherry'] for i, fruit in enumerate(fruits): print(i, fruit) # 0 apple / 1 banana / 2 cherry
💡 生活类比 老师点名时，不只喊"张三"，而是喊"第 1 号，张三；第 2 号，李四……" —— enumerate() 就是帮你自动加上那个序号的工具人，再也不用自己数了。

这节课在 Python 里是干什么用的？

L13 学的是"主动 raise（抛出）exception"——你自己说哪里有问题。这节课学的是"捕获（catch）exception"——当有错误发生时，不让程序崩掉，而是优雅地处理它。想象你写了一个让用户输入数字的程序，用户输入了"abc"——你不可能阻止用户犯蠢，但你可以提前设好"容错机制"，让程序在出错时不崩溃，而是告诉用户"请重新输入"。这就是 try/except 的意义。

Try / Except 基本结构 🧠 理解逻辑 ⭐ 必掌握
💡 生活类比 Try/except 就像开车时的安全气囊：
try 块 = 正常开车（做你想做的事）
except 块 = 撞车时安全气囊弹出（出错时做的补救）
有了安全气囊，撞车不一定会死人；有了 except，程序报错不一定会崩溃。
try: s = input('Please enter a number: ') number = int(s) # ← 这行可能会崩（用户输入 "abc" 的话） print('No error, you entered:', number) except ValueError: print('That was not a valid number!') # ← 只有 ValueError 发生时才执行

执行流程：Python 先跑 try 块；如果抛出了 ValueError，立刻跳到 except ValueError 块执行；如果没有报错，except 块就被跳过。

⚠️ 什么时候该用 try/except？只用来处理"你没法提前防止"的错误！ 比如用户输入的内容、网络连接、文件存不存在——这些你作为程序员控制不了。
绝对不要用 try/except 来"掩盖"你自己代码里的 bug——那样 bug 永远找不到！
处理多种 Exception 📝 记住语法
# 方式 1：分开写多个 except（推荐，每种错误处理方式不同） try: result = int(input('Enter index: ')) my_list = [10, 20, 30] print(my_list[result]) except ValueError: print('Input is not a number') except IndexError: print('Index out of range') # 方式 2：用 tuple 把多个 exception 合并在一个 except 里 try: # ... some code ... pass except (IndexError, TypeError): print('IndexError or TypeError occurred') # 方式 3：裸 except（捕获所有错误——一般不推荐！） try: pass except: print('Something went wrong') # 藏了什么错都不知道，慎用！
finally 块：无论如何都执行 🧠 理解逻辑

finally 块里的代码无论如何都会运行，不管 try 成功还是 except 被触发，甚至就算里面有 return、break、continue 也照样跑。

while True: try: s = input('Please enter a valid number: ') number = int(s) break # ← 成功了，跳出 while 循环 except ValueError: print('Invalid input!') finally: print('This always prints.') # ← 不管成功还是失败，每次都打印 print("Valid number entered: " + str(number))
💡 生活类比 finally 就像餐厅关门前的"收拾桌子"——不管今天生意好不好，客人满不满意，关门前总要收拾桌子。在文件操作里，finally 常用来保证 close() 一定被执行，不管读取过程有没有报错。

这节课在 Python 里是干什么用的？

到目前为止，你用的所有数据类型（int、list、str）都是 Python 内置的。OOP（Object-Oriented Programming，面向对象编程）让你能自己定义新的数据类型。这是编程的一次重大升级：你可以创建一个 "Student" 类型，里面存名字、学号、成绩，还能有"查成绩""改名字"这些操作——远比单独用 list 和 dict 管理要清晰得多。几乎所有大型软件都是用 OOP 写的。

什么是 Object-Oriented Programming？ 🧠 理解概念
💡 生活类比 想想乐高积木。乐高工厂设计了一种"小人"零件的模具（Class/Blueprint，蓝图），规定了小人有头、身体、手、腿，还能"坐下"和"拿东西"。工厂用这个模具可以批量生产无数个小人——每个小人都是一个Object（对象/实例）。每个小人的颜色、头发可以不同，但结构都遵循同一个模具。

Python 里：Class（类）= 模具，Object（对象）= 用模具做出来的具体东西。

你其实从第一天就在用 OOP——只是没意识到：

a = "hello" # a 是 str 类的一个 object（实例） a.upper() # upper() 是 str 这个 class 里定义的 method（方法） my_list = [1, 2, 3] # my_list 是 list 类的一个 object my_list.append(4) # append() 是 list 这个 class 里定义的 method print(type(a)) # <class 'str'> ← 每个东西都属于某个 class！
定义一个 Class 📝 记住语法 ⭐ 必掌握
# 定义一个新的 class（蓝图） class MyClass: """a new type""" # docstring 还是要写的 # 用 class 创建一个 object（实例） my_object = MyClass() # 调用 class 名就能创建，就像调用函数一样 print(type(my_object)) # <class '__main__.MyClass'> ← '__main__' 指的是当前文件

关键区别：

  • Class（类）：设计图 / 模具，本身不是具体东西
  • Object / Instance（对象 / 实例）：用 class 造出来的具体东西，MyClass() 就会创建一个
⚠️ class 名首字母一定大写！ Python 惯例：class 名用 CamelCase（驼峰命名），如 Student、BankAccount；变量和函数名用 snake_case（下划线），如 my_student、get_balance。这是代码可读性的基本规范。
Attributes（属性）：给 Object 贴数据 🧠 理解逻辑 ⭐ 必掌握

Attribute 就是"属于这个 object 的变量"。通过 object.attribute_name 访问和修改：

class MyClass: """a new type""" my_object = MyClass() # 给 object 设置 attribute my_object.name = "John" my_object.age = 20 # 读取 attribute print(my_object.name) # John print(my_object.age) # 20 # 修改 attribute my_object.name = "Max" print(my_object.name) # Max
💡 生活类比 Attribute 就像一个人的"个人信息档案"里的字段：姓名、年龄、学号。每个 object 都有自己的一套，互不干扰。my_object.name 就是说"这个 object 的 name 字段"，用点号 . 表示"属于谁的"。

这节课在 Python 里是干什么用的？

上节课知道了 class 是什么，这节课教你怎么把 class 写得"专业"：用 __init__ 让每个 object 一生下来就有自己的数据；用 methods（方法）给 object 附上能做的事；用 __str__ 控制 object 打印出来长什么样。学完这两节 OOP，你就能理解为什么 my_list.append() 是这样用的，以及未来你自己写的类型也能这么用。

Constructor（构造函数）：__init__ 🧠 理解逻辑 ⭐ 必掌握

每次你写 MyClass() 创建一个 object，Python 会自动调用一个叫 __init__ 的特殊方法——这就是 constructor（构造函数）。你可以自己定义它，控制 object 创建时长什么样：

class Student: def __init__(self, name, ID=0): # self 代表"正在被创建的这个 object 本身" self.name = name # 把传进来的 name 存成 attribute self.ID = ID # ID 有默认值 0 # 创建 object 时传参数，__init__ 会自动接收 john = Student("John", 20) print(john.name, john.ID) # John 20 # 有默认值，可以少传一个 default_student = Student("no name") print(default_student.name, default_student.ID) # no name 0
💡 生活类比 __init__ 就像新生入学填的注册表。每个学生（object）一入学（被创建），学校（class）就要求你填上姓名、学号。这些信息从此就跟着你这个人（object）了，其他地方随时可以查。

关于 self 的关键理解：

  • self 是约定俗成的名字（理论上可以叫其他的，但绝对不要这么做）
  • self 代表"当前这个 object 本身"。在 class 内部，你不知道 object 未来会叫什么名字，所以用 self 代替
  • 定义 __init__ 时 self 必须是第一个参数；但调用时你不用传——Python 自动传入
  • __init__ 没有 return 语句（不需要，也不应该有）
⚠️ 忘传必填参数会报 TypeError 如果 __init__(self, name, ID) 两个参数都没有默认值，Student() 不传参数会报 TypeError: __init__() missing 2 required positional arguments。注意 self 不算！
Class Attributes vs Instance Attributes 🧠 理解逻辑 ⭐ 容易混淆！
Instance Attribute（实例属性） 在 __init__ 里用 self.xxx 定义。
每个 object 有自己独立的一份，互不影响。
Class Attribute（类属性） 在 class 最顶部定义（不在任何方法里）。
所有 object 共享同一个值。
class Car: fuel = 100 # ← Class Attribute：所有 Car 对象共享 def __init__(self, brand): self.brand = brand # ← Instance Attribute：每辆车自己的 car1 = Car("Toyota") car2 = Car("BMW") print(car1.fuel) # 100 ← 从 class 继承来的 print(car2.fuel) # 100 car1.fuel = 20 # 给 car1 单独设置 → 会创建一个 instance attribute，遮住 class attribute print(car1.fuel) # 20 ← car1 自己的 print(car2.fuel) # 100 ← car2 不受影响，还是看 class attribute
⚠️ 修改 Class Attribute 的正确方式 如果你想真正改掉所有 object 共享的值，要用 Car.fuel = 50（通过 class 名修改），而不是 car1.fuel = 50（那只是给 car1 创建了一个独立的 instance attribute）。
Methods（方法）：属于 Class 的函数 🧠 理解逻辑 ⭐ 必掌握

Method 就是定义在 class 里面的 function。和普通 function 的唯一区别：第一个参数必须是 self（代表调用这个方法的那个 object）。

class Student: def __init__(self, name, ID=0): self.name = name self.ID = ID def print_info(self): # 只需要 self，无其他参数 print("Name:", self.name, "ID:", self.ID) def modify_name(self, new_name): # self + 额外参数 self.name = new_name # 修改的是"这个 object"的 name john = Student("John", 20) john.print_info() # Name: John ID: 20 john.modify_name("John Doe") john.print_info() # Name: John Doe ID: 20

调用方法时，self 是自动传的：john.print_info() 等价于 Python 内部执行 Student.print_info(john)——Python 自动把 john 作为 self 传进去。

__str__()：控制 print() 的输出格式 📝 记住用法 ⭐ 必掌握

默认情况下，print 一个自定义 object 会显示很难看的内存地址：

john = Student("John", 20) print(john) # <__main__.Student object at 0x0000024F6EBC8F10> ← 一堆内存地址，完全没用

定义 __str__ 方法，就能让 print 显示你想要的内容：

class Student: def __init__(self, name, ID=0): self.name = name self.ID = ID def __str__(self): # 注意：必须 return 一个 string，不能 print！ return "Student: " + self.name + ", ID: " + str(self.ID) john = Student("John", 20) print(john) # Student: John, ID: 20 ← 现在看起来像个人了！ print(str(john)) # 同上，str() 也会调用 __str__
💡 生活类比 每个人在不同场合有不同的"自我介绍"。__str__ 就是你教这个 object "当别人 print 你的时候，你怎么介绍自己"。没有定义就是报出一串身份证内存地址，有了 __str__ 就能说"我是 John，学号 20"。
特殊方法触发时机必须 return
__init__(self, ...)创建 object 时自动调用❌ 无 return
__str__(self)print(obj) 或 str(obj) 时调用✅ 必须 return string
